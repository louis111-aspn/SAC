SAC Story 필터와 변수(Variable)

같은 Story라도 보는 사람마다 보고 싶은 기간·조직·버전이 다릅니다. 필터와 Variable을 어디에 배치하고 어떻게 설계하느냐에 따라, 화면이 "유연한 도구"가 될지 "복잡한 미로"가 될지가 결정됩니다.

1. 왜 필요한가 2. 전역 vs 위젯 필터 3. Variable 활용 4. 배치 Before/After 5. 체크리스트 6. 정리 퀴즈
01

왜 필터·변수 설계가 중요한가

필터는 "데이터를 좁히는 기능"이지만, 잘못 설계되면 보는 사람이 화면의 의미를 매번 다시 확인해야 하는 부담을 만듭니다.

흔한 장면

SAC Story를 열었는데 화면 곳곳에 필터 드롭다운이 7~8개 흩어져 있습니다. 어떤 필터는 화면 전체에 영향을 주고, 어떤 필터는 특정 차트에만 영향을 줍니다. 보는 사람은 "지금 이 숫자가 어떤 조건으로 필터링된 결과인지" 매번 필터들을 하나씩 확인해야 합니다.

필터/Variable의 개수 자체보다, 어떤 필터가 "전체에 영향을 주는 기준"이고 어떤 필터가 "특정 위젯의 보조 옵션"인지 구분되어 있는가가 더 중요합니다.

핵심
필터·Variable 설계의 목표는 "선택할 수 있는 것을 늘리는 것"이 아니라, "이 화면이 지금 어떤 조건을 보고 있는지를 한눈에 알 수 있게 하는 것"입니다. Episode 1의 정보 위계 원칙과 마찬가지로, 필터에도 위계가 필요합니다.
02

전역 필터 vs 위젯별 필터

SAC Story의 필터는 적용 범위에 따라 크게 두 가지로 나뉩니다. 이 둘을 혼용하면 "왜 이 차트만 다른 기간을 보여주는가"와 같은 혼란이 발생합니다.

기준전역 필터 (Story / Page Filter)위젯별 필터 (Widget Filter)
적용 범위 Story 전체 또는 Page 내 모든 위젯에 동일하게 적용 특정 위젯에만 적용, 다른 위젯에는 영향 없음
적합한 대상 보고 기간, 회계연도, 버전(Actual/Plan/Forecast), 조직 단위 등 — "이 화면 전체가 어떤 조건을 보고 있는가"를 정의하는 차원 특정 차트에서만 의미 있는 세부 조건 (예: 특정 차트의 Top N 표시, 특정 계정 그룹만 별도로 보는 보조 차트)
배치 위치 화면 상단(헤드라인 영역 인근) — Episode 1의 Z패턴 첫 시선 영역 해당 위젯 내부 또는 바로 옆 — "이 차트에만 적용된다"는 것이 시각적으로 드러나는 위치
위험 신호 전역 필터가 너무 많아 한 화면을 가득 채움 위젯별 필터가 전역 필터처럼 화면 상단에 다른 전역 필터들과 나란히 배치되어 구분이 안 됨
⊘ 구분 없는 배치
  • "기간", "버전", "조직", "Top N 표시", "정렬 기준" 등 7~8개 필터가 화면 상단에 동일한 모양으로 나열
  • 보는 사람은 어떤 필터가 전체에 영향을 주는지 알 수 없음
  • 위젯별 필터를 바꿔도 다른 차트의 숫자가 그대로라 "오류"로 오인될 수 있음
✓ 위계가 있는 배치
  • 전역 필터(기간/버전/조직) 3개만 화면 최상단에 강조 배치
  • 위젯별 보조 옵션(Top N, 정렬 등)은 해당 위젯 우상단의 작은 컨트롤로 배치
  • "전체 조건"과 "이 차트만의 옵션"이 위치와 크기로 구분됨
03

Variable을 활용한 동적 화면 구성

Variable(변수)은 사용자의 선택(기간, 버전, 조직 등)에 따라 화면의 데이터와 라벨이 함께 바뀌도록 만드는 기능입니다. 아래에서 직접 선택해보면서 동작을 확인해보세요.

SAC 적용 팁
위 예시처럼 Variable을 사용하면 KPI 카드의 제목 라벨과 값이 함께 바뀝니다. 이 두 가지가 항상 같이 움직이도록 설계해야, 보는 사람이 "지금 보고 있는 숫자가 무엇에 대한 것인지" 혼동하지 않습니다. 라벨 없이 값만 Variable로 바뀌는 경우, 화면을 캡처해서 공유하면 어떤 조건의 데이터인지 알 수 없게 됩니다.
자주 하는 실수
Variable의 기본값(Default)을 설정하지 않으면, Story를 처음 여는 사람은 빈 화면이나 전체 데이터를 보게 되어 "무엇을 봐야 하는지" 알기 어렵습니다. 가장 최근 마감 기간, 가장 많이 조회되는 조직 단위 등을 기본값으로 지정해, 첫 화면 자체가 의미 있는 "헤드라인"이 되도록 구성합니다.

05

필터·변수 체크리스트

필터와 Variable을 추가하기 전, 아래 항목으로 점검합니다.

  • 이 필터는 "화면 전체의 조건"인가, "특정 위젯만의 옵션"인가? 전역 필터와 위젯별 필터가 구분되어 있는가?
  • 전역 필터는 화면 상단(헤드라인 영역 인근)에 모여 있고, 개수가 3~5개를 넘지 않는가?
  • Variable에 기본값(Default)이 설정되어 있어, Story를 처음 열었을 때도 의미 있는 화면이 보이는가?
  • Variable 값이 바뀔 때 KPI 카드의 라벨과 값이 함께 바뀌도록 구성되어 있는가?
  • 위젯별 보조 옵션(Top N, 정렬, 단위 등)은 해당 위젯과 시각적으로 가깝게 배치되어 있는가?
