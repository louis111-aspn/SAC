SAC Story 레이아웃과 위젯 구조

Episode 1~2에서 다룬 정보 위계, 시선 흐름, 차트 선택 원칙이 실제 SAC Story에서는 Page / Section / Widget이라는 구조와 Responsive / Canvas 모드를 통해 구현됩니다. 이번 Episode는 그 구조를 다룹니다.

01

Page / Section / Widget 구조 이해

SAC Story는 Page → Section(Responsive 모드의 경우 Column/Row) → Widget의 계층 구조를 가집니다. 이 구조는 Episode 1의 "정보 위계(헤드라인 → 근거 → 디테일)"를 물리적으로 담는 그릇입니다.

계층 구조 매핑

Page
하나의 보고 주제 단위 (예: "월간 비용 분석"). 여러 Section을 포함하며, 화면 탭으로 구분되는 최상위 단위입니다.
Section A — 헤드라인 영역
Episode 1의 "1단계: 핵심 메시지"에 대응
Widget: KPI Card (B사업부 비용 +12.3%)
Section B — 근거 영역
Episode 1의 "2단계: 근거"에 대응
Widget: Bar Chart (계정별 Variance)
Section C — 디테일 영역 (기본 접힘)
Episode 1의 "3단계: 디테일"에 대응 — Section 4 드릴다운 참고
Widget: Table (원장 상세)
핵심
위젯을 먼저 만들고 나중에 배치를 고민하는 것이 아니라, Section 단위로 "이 영역은 정보 위계의 몇 단계인가"를 먼저 정의한 뒤 그 안에 위젯을 채우는 순서로 작업하면 Episode 1~2의 원칙이 구조적으로 지켜집니다.

용어 정리

  • Page: Story 내 탭 단위. 하나의 보고 목적(주제)에 대응.
  • Section: Page 내 가로 영역 구분 단위 (Responsive 모드 기준). 정보 위계의 한 "단(段)"에 대응시키는 것을 권장.
  • Widget: Chart, Table, KPI(Numeric Point), Filter, Text 등 실제 화면 요소.
02

Responsive vs Canvas 모드 선택 기준

SAC Story는 두 가지 레이아웃 모드를 제공합니다. 어떤 모드를 선택하느냐에 따라 Episode 1의 "Grid Alignment(정렬·여백)" 원칙을 지키는 난이도와 방식이 달라집니다.

기준Responsive 모드Canvas 모드
배치 방식 Section/Column 단위로 자동 배치, 화면 크기에 따라 줄바꿈·재배열됨 좌표(X, Y) 기반 절대 배치, 픽셀 단위로 고정
화면 크기 대응 PC/태블릿/모바일 등 다양한 화면에서 자동 최적화 지정한 해상도에서는 정확하지만, 다른 화면 크기에서는 잘림·여백 발생 가능
정렬·여백 정밀도 Section 내 위젯 간격은 균등하게 자동 정렬되나, 미세 조정은 제한적 Episode 1~2에서 강조한 "픽셀 단위 정렬"을 직접 구현 가능 — PPT의 Grid Alignment 원칙과 유사
적합한 용도 다양한 기기에서 열람되는 운영 대시보드, 정기 보고용 Story 특정 화면(회의실 모니터, 발표용 단일 화면)에 맞춘 정밀 레이아웃, 인쇄·PDF 출력용 보고서형 Story
유지보수 위젯 추가·삭제 시 레이아웃이 자동 재배열되어 관리 부담 낮음 위젯 추가·삭제 시 다른 요소와의 좌표 충돌을 수동으로 재조정 필요
선택 가이드
"누가, 어떤 화면으로 보는가"가 기준입니다. 여러 사용자가 PC·모바일 등 다양한 환경에서 보는 운영용 Story라면 Responsive를, 임원 보고용으로 한 화면에 픽셀 단위로 정렬된 "장표"에 가까운 결과물이 필요하다면 Canvas를 선택합니다. 하나의 Story 안에서 Page별로 모드를 다르게 구성하는 것도 가능합니다.
자주 하는 실수
Canvas 모드에서 "일단 보기 좋게" 위젯을 자유롭게 배치하다 보면, Section 간 간격이 13px, 17px, 22px처럼 제각각이 되는 경우가 많습니다. PPT 작업에서 Gutter를 15~20pt 단위로 고정하는 것과 동일하게, Canvas 모드에서도 위젯 간격을 일정 단위(예: 16px)로 고정하는 규칙을 미리 정해두어야 합니다.

04

Linked Analysis를 활용한 드릴다운 설계

기본 화면은 "헤드라인 + 근거"만 가볍게 보여주고, "디테일"은 클릭(Linked Analysis · 드릴다운)으로 접근하도록 설계하면 Episode 1의 정보 위계 원칙을 화면 인터랙션으로 확장할 수 있습니다.

SAC 적용 팁
SAC Story에서는 Table/Chart 위젯에 Linked Analysis를 설정해, 한 위젯에서의 선택(필터)이 같은 Page의 다른 위젯에도 반영되도록 구성할 수 있습니다. 예를 들어 KPI Row에서 "B사업부"를 클릭하면 하단 Chart와 Table이 B사업부 데이터로 자동 필터링되는 방식입니다. 이렇게 하면 별도 "디테일 Section"을 항상 펼쳐두지 않고도, 필요한 사람만 클릭해서 디테일에 접근하게 됩니다.
자주 하는 실수
모든 디테일을 Linked Analysis 없이 별도의 "상세 Page"로 분리하는 경우, 보는 사람이 탭을 옮겨 다니며 맥락을 잃기 쉽습니다. 핵심 메시지와 직접 연결된 디테일은 같은 Page 내 드릴다운으로, 완전히 다른 분석 주제의 디테일은 별도 Page로 분리하는 것이 기준입니다.
05

SAC 레이아웃 체크리스트

Story 초안을 완성한 뒤, 아래 항목으로 레이아웃을 점검합니다.

  • 각 Section이 정보 위계(헤드라인 / 근거 / 디테일) 중 어느 단계에 해당하는지 명확한가?
  • Page 내 Section의 배치 순서가 의도한 시선 흐름(Episode 1의 Z/F 패턴)과 일치하는가?
  • Responsive·Canvas 모드를 선택한 이유(열람 환경)가 명확한가?
  • Canvas 모드 사용 시, 위젯 간 간격이 일정 단위로 고정되어 있는가?
  • 디테일 데이터(원장 등)가 기본 화면에 항상 노출되어 있지 않고 드릴다운으로 접근 가능한가?
  • Linked Analysis를 적용한 위젯과 그렇지 않은 위젯이 사용자에게 구분되어 인지되는가? (예: 클릭 가능 표시)
