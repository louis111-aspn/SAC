Module 1

왜 보고서는 읽기 어려운가

경영진이 실제로 보고서에서 찾는 정보와, 우리가 만드는 보고서 사이의 간극을 이해합니다.

1-1. 경영진은 숫자를 보지 않는다

경영진이 보고서를 펼칠 때 실제로 찾는 것은 절대값 그 자체가 아닙니다. 다음 네 가지를 빠르게 캐치하고자 합니다.

변화 (Change)
전기 대비, 계획 대비 무엇이 달라졌는가
예외 (Exception)
정상 범위를 벗어난 항목은 무엇인가
위험 (Risk)
향후 계획 달성에 위협이 되는 요소는 무엇인가
원인 (Cause)
왜 이런 결과가 나왔는가
⚠ 숫자 나열형 보고서의 한계

100페이지 경영보고서, ERP 출력물, 전통 BI 리포트는 데이터를 빠짐없이 담고 있지만, 위 4가지 질문에 즉시 답하지 못하는 경우가 많습니다. 정보의 양과 의사결정 속도는 반비례 관계에 놓이기 쉽습니다.

Module 2

IBCS란 무엇인가

국가·기업마다 서로 다른 보고 방식을 하나의 언어로 통일하려는 국제 표준입니다.

2-1. IBCS의 탄생 배경

International Business Communication Standards Association은 다음과 같은 문제를 해결하기 위해 만들어졌습니다.

  • 국가마다 다른 보고 방식
  • 회사마다 다른 색상 체계
  • 동일한 수치를 표현하는 서로 다른 방식
✕ 결과

표준화되지 않은 보고서는 해석 오류와 의사결정 지연으로 이어집니다. 동일한 데이터라도 색상, 정렬, 단위 표기 방식이 다르면 읽는 사람마다 다른 결론에 도달할 수 있습니다.

2-2. IBCS 핵심 목표

"모든 보고서를 동일한 언어로 표현하자"

Say — 메시지를 먼저 정의
차트나 표를 그리기 전에, 이 페이지가 전달하고자 하는 핵심 메시지(결론)를 한 문장으로 먼저 정의합니다. 슬라이드 제목은 이 메시지를 그대로 담은 완전한 문장이어야 합니다.
Unify — 표현 방식 통일
색상, 단위, 정렬, 축의 스케일 등 시각적 문법을 전사적으로 동일하게 적용합니다. 동일한 의미는 항상 동일한 형태로 표현되어야 합니다.
Condense — 정보 압축
한 페이지에 너무 많은 정보를 담지 않습니다. 핵심 KPI와 그 변화량을 중심으로 정보를 압축해 한눈에 파악할 수 있도록 합니다.
Check — 시각화 품질 검증
완성된 보고서가 처음 보는 사람에게도 오해 없이 전달되는지 검증합니다. 색상, 차트 종류, 축 스케일이 메시지를 왜곡하지 않는지 확인합니다.
Express — 핵심 강조
가장 중요한 데이터 포인트(예외, 위험 요소)를 색상이나 강조 표시를 통해 시각적으로 부각합니다. 강조는 의미가 있을 때만 사용합니다.
Simplify — 불필요 요소 제거
그림자, 입체 효과, 불필요한 그리드라인, 장식용 아이콘 등 정보 전달에 기여하지 않는 모든 요소를 제거합니다.
Structure — 논리적 구성
페이지 내 요소들을 KPI → 원인분석 → 액션 순서로 논리적으로 배치하여, 읽는 사람의 시선 흐름이 자연스럽게 결론으로 이어지도록 합니다.
Module 3

IBCS가 권장하는 시각화 원칙

색상, 차트 종류, 그리고 Variance(차이) 중심 사고 — 시각화의 세 가지 핵심 규칙입니다.

3-1. 색상은 장식이 아니다

색상은 데이터의 종류(시나리오)를 구분하는 의미 체계로만 사용되어야 합니다.

✅ 권장
Actual = 검정
Plan = 회색
Forecast = 점선

시나리오별로 고정된 색상/패턴을 일관되게 사용 → 어떤 페이지를 보아도 동일하게 해석 가능

❌ 비권장
무지개 차트

의미 없는 컬러 사용은 시선을 분산시키고, 어떤 색이 무엇을 의미하는지 매번 범례를 확인해야 하는 인지 부하를 유발합니다.

3-2. 차트 선택 기준

Module 4

SAC 디자인 철학 이해하기

SAP Analytics Cloud의 최신 기능들이 IBCS 원칙과 어떻게 맞닿아 있는지 살펴봅니다.

4-1. SAC가 추구하는 리포트 방향

SAP는 Analytics를 다음 세 단계의 연속된 흐름으로 정의합니다.

Monitor
→
Analyze
→
Decide

단순히 데이터를 시각화하는 것을 넘어, 모니터링 → 원인 분석 → 의사결정으로 이어지는 사용자 여정을 설계하는 것이 SAC의 목표입니다.

4-2. SAC 기능과 IBCS 매핑

IBCS 원칙SAC 기능
SayDynamic Text
UnifyCorporate Styling (Theme Designer)
CondenseKPI Tile
ExpressSemantic Color (Threshold/Status)
SimplifyOptimized Story Experience (불필요 위젯 제거)
CheckStory Validation / Responsive Layout
StructurePage Layout (Grid / Canvas)

4-3. SAC 표준 콘텐츠 분석

SAP Best Practice Dashboard의 일반적인 구성요소는 다음과 같습니다.

KPI Header
핵심 지표를 페이지 상단에 압축 배치 — Condense
Variance Area
계획·전기 대비 차이를 별도 영역으로 노출 — Express
Trend Analysis
시계열 추이를 통해 패턴과 예외를 파악 — Analyze
분석: IBCS 원칙이 어디에 적용되었는가?

SAP Best Practice Dashboard에서 KPI Header는 한눈에 핵심 수치를 압축 전달(Condense)하고, Semantic Color로 정상/주의/위험 상태를 즉시 식별(Express)할 수 있게 합니다. Variance Area는 계획 대비 차이를 별도 강조하여 Variance 중심 사고를 구현하며, 전체 페이지 구조는 KPI → Variance → Trend → 원인분석 순서로 배치되어 Structure 원칙을 따릅니다. 이는 IBCS의 SUCCESS 원칙이 별도의 이론 교육 없이도 SAC 표준 콘텐츠 안에 이미 내재화되어 있음을 보여줍니다.

5-4. Executive Dashboard 설계 실습 — Before / After

Before — 화려한 대시보드
  • 파이차트, 도넛차트, 게이지 차트 혼합
  • 위젯마다 다른 색상 테마
  • KPI 20개 이상이 한 화면에 나열
  • Variance 정보 없음 (절대값만 표시)
After — IBCS 기반 Dashboard
  • 핵심 KPI 3~5개 + Variance 명시
  • Actual=검정, Plan=회색, Forecast=점선으로 통일
  • KPI → Trend → 원인분석 순서로 배치
  • Semantic Color는 임계치 초과 항목에만 적용
