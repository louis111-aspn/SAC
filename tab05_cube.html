1. Cube View의 역할과 설계 원칙

Cube View는 트랜잭션 데이터(금액·수량·건수)를 집계하고, Tab 03·04에서 만든 Dimension View와 Foreign Key Association으로 연결하는 중간 집계 계층입니다. Query View(Tab 06)가 이 Cube를 소스로 참조하여 SAC에 노출합니다. Cube View 자체는 SAC Live Model에 직접 노출되지 않습니다.

항목 Cube View (#CUBE) Query View (@Analytics.query: true)
역할 Measure 집계 + Dimension FK 연결 SAC Live Model의 직접 노출 대상
Measure 정의 @DefaultAggregation으로 집계 방식 선언 Restricted/Calculated Measure 추가 정의
Variable/Parameter 파라미터 선언 가능 (선택) Variable 선언 및 SAC 필터 연동
Dimension 참조 @ObjectModel.foreignKey.association으로 연결 Cube의 Dimension을 그대로 상속
SAC 노출 직접 노출 안 됨 직접 노출 — OData 서비스 등록 대상
ASPN 예시 ZASPN_C_FinancialCube ZASPN_C_FinancialQuery (Tab 06)
설계 철학 — Cube와 Query의 분리

Cube View에는 재사용 가능한 Measure와 Dimension 참조만 담습니다. 사용자 목적에 따라 다른 Query View(P&L Query, B/S Query, Cost Query 등)가 하나의 Cube를 공유하여 소비합니다. Cube를 Query와 분리하면 Measure 변경 시 모든 Query에 자동 반영되어 유지보수성이 높아집니다.

2. Measure 선언 — @DefaultAggregation

Cube View에서 숫자 필드를 Measure로 만들려면 @DefaultAggregation Annotation을 선언합니다. 이 Annotation이 없으면 Analytics Engine은 해당 필드를 Dimension으로 간주합니다.

가장 많이 사용
#SUM
@DefaultAggregation: #SUM
합계 집계. 금액(Amount), 수량(Quantity) 등 대부분의 재무·원가 Measure에 사용합니다. ASPN Group의 AmountInCompanyCodeCurrency, AmountInGlobalCurrency 모두 #SUM.
건수 집계
#COUNT_DISTINCT_NULLS / #COUNT
@DefaultAggregation: #COUNT_DISTINCT_NULLS
전표 건수, 계정 수 등 중복 제거 건수 집계에 사용합니다. #COUNT는 NULL 포함 전체 행 수, #COUNT_DISTINCT_NULLS는 NULL을 하나로 처리합니다.
잔액·재고
#MIN / #MAX
@DefaultAggregation: #MIN
시작 잔액, 최저가, 최고가 등에 사용합니다. 재무상태표 잔액처럼 합산이 아닌 특정 시점 값이 필요한 경우 Query View의 Exception Aggregation(Tab 06)과 함께 활용합니다.
집계 불가 Measure
#FORMULA / #NOP
@DefaultAggregation: #NOP
#NOP(No Operation): 집계를 Analytics Engine에 위임합니다. Cube에서 집계 방식을 고정하지 않고 Query View의 Calculated Measure에서 결정하고 싶을 때 사용합니다. 남용 시 SAC에서 집계 결과가 예상과 다를 수 있으므로 주의합니다.

@Aggregation.default vs. @DefaultAggregation

주의 — 두 Annotation의 차이

@DefaultAggregation: #SUM은 필드 레벨에 선언하며, 이 필드의 기본 집계 방식을 지정합니다.
@Aggregation.default: #SUM은 구형 구문으로 동일한 역할을 하지만, 신규 개발에서는 @DefaultAggregation 사용을 권장합니다.
두 Annotation이 동일 필드에 함께 선언되면 @DefaultAggregation이 우선합니다.

3. Unit/Currency Semantics — @Semantics 선언

SAC에서 금액·수량 Measure를 올바르게 표시하려면 해당 필드가 어떤 통화 코드·단위 필드와 연결되는지 @Semantics Annotation으로 명시해야 합니다. 이 선언이 없으면 SAC에서 통화 변환·단위 표시가 불가능합니다.

Annotation 적용 대상 설명
@Semantics.amount.currencyCode: 'Currency' 금액 필드 이 금액 필드의 통화 코드를 보관하는 필드명을 지정. SAC가 통화 Dimension과 자동 연결
@Semantics.currencyCode: true 통화 코드 필드 이 필드가 통화 코드임을 선언. @Semantics.amount가 참조하는 필드
@Semantics.quantity.unitOfMeasure: 'Unit' 수량 필드 이 수량 필드의 단위 코드를 보관하는 필드명을 지정
@Semantics.unitOfMeasure: true 단위 코드 필드 이 필드가 단위 코드(EA, KG, M 등)임을 선언
@Semantics.fiscal.year: true 회계연도 필드 회계연도 Dimension 인식. SAC 시간 필터와 연동
@Semantics.fiscal.period: true 회계기간 필드 회계기간(월) Dimension 인식
@Semantics.fiscal.yearPeriod: true 연기간 결합 필드 회계연도+기간 결합 필드. SAC 시계열 분석의 기준 필드
ASPN Group — 다중 통화 Semantics 설계

ASPN Group은 ASPN Korea(KRW)와 ASPN Europe(EUR)이 혼재합니다. Cube View에서는 회사 코드 통화(LC)와 그룹 통화(GC: KRW)를 각각 선언합니다. AmountInCompanyCodeCurrency는 CompanyCodeCurrency 필드를, AmountInGlobalCurrency는 고정값 'KRW'를 통화 코드로 연결합니다.

4. Dimension Foreign Key Association 연결

Cube View의 키 필드와 Tab 03에서 만든 Dimension View를 @ObjectModel.foreignKey.association으로 연결합니다. 이 Annotation이 선언되어야 SAC가 해당 키 필드를 Dimension으로 자동 인식합니다.

통화 코드(CompanyCodeCurrency)와 단위(BaseUnit) 필드도 Dimension입니다. SAP 표준 I_Currency, I_UnitOfMeasure를 @Analytics.dimensionType: #MEASURE_DIMENSION으로 참조하면 SAC가 통화·단위를 기준으로 데이터를 분리 표시합니다. ASPN Europe의 EUR 금액과 ASPN Korea의 KRW 금액이 통화 Dimension으로 구분됩니다.

5. 완성 예시 — ZASPN_C_FinancialCube

ASPN Group 연결재무제표 분석을 위한 Cube View 전체 코드입니다. ACDOCA(Universal Journal) 기반 소스, 다중 통화 Measure, Dimension FK 연결, Unit/Currency Semantics를 모두 포함합니다.

ABAP CDS · DDL ZASPN_C_FinancialCube — Cube View 전체 해부
1–8
@Analytics.dataCategory: #CUBE @VDM.viewType: #CONSUMPTION @EndUserText.label: 'ASPN Financial Cube' @AccessControl.authorizationCheck: #CHECK @ObjectModel.usageType: { serviceQuality: #A, sizeCategory: #XXL, dataClass: #MIXED }
View 레벨 필수 Annotation. #CUBE: Analytics Cube 선언. @ObjectModel.usageType: 서비스 품질·데이터 규모 힌트. ACDOCA 기반 Cube는 #XXL(대용량). SAP 표준 가이드라인 준수.
9–13
define view entity ZASPN_C_FinancialCube as select from I_JournalEntryItem as JE // ── Dimension FK Associations ── association [0..1] to ZASPN_I_CompanyCode as _CompanyCode on $projection.CompanyCode = _CompanyCode.CompanyCode
소스 및 Association 선언. 소스는 SAP 표준 Interface View I_JournalEntryItem. ACDOCA를 직접 참조하지 않고 표준 Interface View를 통해 안정성을 확보합니다.

association [0..1] to ZASPN_I_GLAccount as _GLAccount on $projection.ChartOfAccounts = _GLAccount.ChartOfAccounts and $projection.GLAccount = _GLAccount.GLAccount association [0..1] to ZASPN_I_ProfitCenter as _ProfitCenter on $projection.ControllingArea = _ProfitCenter.ControllingArea and $projection.ProfitCenter = _ProfitCenter.ProfitCenter association [0..1] to I_Currency as _Currency on $projection.CompanyCodeCurrency = _Currency.Currency
복합 키 Association. GL 계정은 ChartOfAccounts + GLAccount 복합 키. 이익센터는 ControllingArea + ProfitCenter. 복합 키 Association은 ON 조건에 and로 모든 키를 명시해야 합니다.

{ // ── 키 필드 + FK Association 연결 ── @ObjectModel.foreignKey.association: '_CompanyCode' key JE.CompanyCode, @ObjectModel.foreignKey.association: '_GLAccount' key JE.ChartOfAccounts, @ObjectModel.foreignKey.association: '_GLAccount' key JE.GLAccount, @ObjectModel.foreignKey.association: '_ProfitCenter' key JE.ControllingArea, @ObjectModel.foreignKey.association: '_ProfitCenter' key JE.ProfitCenter, key JE.FiscalYear, key JE.FiscalPeriod, key JE.AccountingDocument, key JE.AccountingDocumentItem
FK Association 선언 위치. @ObjectModel.foreignKey.association은 해당 키 필드 바로 위에 선언합니다. 복합 키는 각 키 필드 위에 동일한 Association 이름을 반복 선언합니다.
41–60
// ── Measure 필드 ── // 회사 코드 통화 금액 (LC) @Semantics.amount.currencyCode: 'CompanyCodeCurrency' @DefaultAggregation: #SUM @EndUserText.label: 'Amount (Local Currency)' JE.AmountInCompanyCodeCurrency as AmountLC, // 그룹 통화 금액 (GC: KRW) — currency_conversion @Semantics.amount.currencyCode: 'GroupCurrency' @DefaultAggregation: #SUM @EndUserText.label: 'Amount (Group Currency KRW)' currency_conversion( amount => JE.AmountInCompanyCodeCurrency, source_currency => JE.CompanyCodeCurrency, target_currency => 'KRW', exchange_rate_date => JE.PostingDate, exchange_rate_type => 'M' ) as AmountGC, // 수량 Measure @Semantics.quantity.unitOfMeasure: 'BaseUnit' @DefaultAggregation: #SUM JE.Quantity as Quantity,
Measure 3종 선언 패턴. ① LC 금액: 통화 코드 필드 참조. ② GC 금액: currency_conversion()으로 KRW 환산. ③ 수량: 단위 코드 필드 참조. 각 Measure마다 @Semantics + @DefaultAggregation 쌍이 필수입니다.
61–80
// ── 통화·단위 코드 필드 (Semantics 선언) ── @Semantics.currencyCode: true @ObjectModel.foreignKey.association: '_Currency' JE.CompanyCodeCurrency, @Semantics.currencyCode: true 'KRW' as GroupCurrency, @Semantics.unitOfMeasure: true JE.BaseUnit, // ── 회계 시간 Dimension ── @Semantics.fiscal.year: true JE.FiscalYear, @Semantics.fiscal.period: true JE.FiscalPeriod, @Semantics.fiscal.yearPeriod: true JE.FiscalYearPeriod, // ── Association 노출 ── _CompanyCode, _GLAccount, _ProfitCenter, _Currency }
통화·단위·시간 Semantics. 그룹 통화 GroupCurrency는 리터럴 'KRW'로 고정 선언. 회계 시간 필드는 @Semantics.fiscal.*으로 SAC 시계열 분석과 연동. 모든 Association을 필드 목록 마지막에 노출합니다.
ACDOCA 직접 참조 vs. I_JournalEntryItem 참조

ACDOCA 테이블을 직접 참조하는 것보다 SAP 표준 Interface View I_JournalEntryItem을 소스로 사용하는 것을 권장합니다. 표준 View는 SAP 업그레이드 시 하위 호환성이 보장되며, 이미 최적화된 필드 매핑과 Annotation이 적용되어 있습니다. ASPN Group 커스텀 필드(Z-필드)가 필요한 경우에만 ACDOCA Extension View를 소스에 추가합니다.

6. Cube View 허용 / 불가 패턴

패턴 설명 및 대안
✓ @DefaultAggregation: #SUM Measure 선언 Cube View의 핵심 패턴. 숫자 필드마다 집계 방식을 Annotation으로 선언합니다.
✓ @ObjectModel.foreignKey.association 키 필드와 Dimension View를 연결하는 핵심 Annotation. SAC Dimension 자동 인식에 필수.
✓ currency_conversion() 인라인 통화 변환 Cube 내에서 그룹 통화 Measure를 직접 계산. SAP 환율 테이블(TCURR) 기반으로 동작.
✓ 파라미터(with parameters) 선언 Cube에 파라미터 선언 가능. Query View에서 파라미터를 전달받아 사용.
✗ @Analytics.query: true 동시 선언 Cube View는 Query View가 아닙니다. 두 Annotation을 동시에 선언하면 오류. Cube에서는 #CUBE만 선언합니다.
✗ Restricted Measure / Calculated Measure 선언 Restricted/Calculated Measure는 Query View에서 선언합니다. Cube에서의 선언은 지원되지 않습니다.
✗ Variable(@AnalyticsDetails.query.variable) 선언 Variable은 Query View 전용입니다. Cube의 파라미터(with parameters)와 다릅니다.
! GROUP BY 명시적 사용 Analytics Engine이 Annotation 기반으로 자동 집계합니다. 명시적 GROUP BY는 일반 CDS View 패턴이며 Cube에서는 Annotation 방식을 사용합니다.
! ACDOCA 직접 참조 기술적으로 가능하지만 SAP 표준 Interface View(I_JournalEntryItem) 사용을 권장합니다. 업그레이드 호환성 문제 발생 가능.
✅ Cube View 배포 전 품질 체크리스트 ▼
