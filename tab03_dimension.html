1. Dimension View의 역할과 위치

Analytics 아키텍처에서 Dimension View는 분석 축(Axis)을 정의합니다. Cube View의 Measure(숫자)를 어떤 기준으로 자르고 볼 것인가—회사 코드별, 계정별, 이익센터별—를 결정하는 마스터 데이터 서술자입니다. SAC에서 필터·행·열에 배치되는 항목이 모두 Dimension입니다.

구분 Dimension View Cube View Analytics Query View

2. Dimension View 기본 구조 및 핵심 Annotation

해부 분석 — ZASPN_I_CompanyCode (회사 코드 Dimension)

ABAP CDS · DDL ZASPN_I_CompanyCode — Dimension View 전체 구조
1
@Analytics.dataCategory: #DIMENSION
핵심 선언. 이 View가 Dimension임을 Analytics Engine에 알립니다. SAC Live Model이 자동으로 Dimension 목록에 포함시킵니다.
2
@Analytics.dimensionType: #STANDARD
Dimension 유형. #STANDARD(일반), #TIME(시간), #MEASURE_DIMENSION(통화·단위). 시간 Dimension은 SAC 시계열 분석과 연동됩니다.
3
@EndUserText.label: 'Company Code'
SAC 표시 레이블. SE63으로 다국어 번역 가능. SAC Dimension 패널에 이 텍스트가 표시됩니다.
4
@VDM.viewType: #BASIC
VDM 계층. 마스터 데이터 Dimension은 주로 #BASIC 또는 #COMPOSITE. Consumption Layer의 Query가 직접 참조하지 않고 Cube를 통해 간접 참조됩니다.
5
@AccessControl.authorizationCheck: #CHECK
권한 체크. Dimension View에도 반드시 #CHECK를 설정합니다. Value Help 조회 시에도 권한 필터가 적용됩니다.
6–9
define view entity ZASPN_I_CompanyCode as select from I_CompanyCode as CC // Text Association 연결 association [0..*] to I_CompanyCodeText as _Text on $projection.CompanyCode = _Text.CompanyCode // Hierarchy Association 연결 (Tab 04) association [0..1] to ZASPN_H_CompanyCode as _Hierarchy on $projection.CompanyCode = _Hierarchy.CompanyCode
Association 선언 블록. Text Association은 [0..*] — 언어별 여러 텍스트 레코드가 존재하기 때문입니다. Hierarchy Association은 Tab 04에서 생성할 Hierarchy View를 참조합니다.
10–22
{ key CC.CompanyCode, // Text Association 연결 Annotation @ObjectModel.text.association: '_Text' CC.CompanyCode as CompanyCode, CC.CountryChartOfAccounts, CC.Currency, CC.FiscalYearVariant, // Hierarchy Association 노출 @ObjectModel.hierarchyAssociation: '_Hierarchy' _Text, _Hierarchy }
필드 목록. @ObjectModel.text.association은 텍스트를 어느 Association에서 가져올지 지정합니다. @ObjectModel.hierarchyAssociation은 SAC가 Hierarchy를 인식하는 핵심 Annotation입니다.

3. Text Association — 마스터 데이터 텍스트 연결

SAC에서 Dimension 값은 코드(1000)가 아닌 텍스트(ASPN Holdings)로 표시되어야 합니다. 이를 위해 Dimension View는 텍스트를 보관하는 Text View와 Association으로 연결하고, @ObjectModel.text.association으로 어느 Association이 텍스트 소스인지 선언합니다.

Text View — SAP 표준 사용 vs. 커스텀 작성

SAP 표준 Text View 우선 사용

SAP S/4HANA는 주요 마스터 데이터에 표준 Text View를 이미 제공합니다. I_CompanyCodeText, I_GLAccountText, I_ProfitCenterText 등이 대표적입니다. 커스텀 Dimension View를 만들 때 이 표준 Text View를 Association 소스로 직접 참조하는 것이 권장 패턴입니다. 커스텀 Text View는 Z-테이블 기반 커스텀 마스터 데이터에만 직접 작성합니다.

ABAP CDS 권장 패턴 — SAP 표준 I_CompanyCodeText 직접 참조 
// 권장: SAP 표준 Text View를 Association 소스로 직접 참조
// I_CompanyCodeText 는 SAP 제공 표준 View (Language·CompanyCode key, @Semantics.text 이미 선언됨)
association [0..*] to I_CompanyCodeText as _Text
    on $projection.CompanyCode = _Text.CompanyCode

커스텀 Text View 직접 작성이 필요한 경우

Z-테이블 기반 커스텀 마스터 데이터처럼 SAP 표준 Text View가 없을 때만 직접 작성합니다. 이 경우 아래 4가지 규칙을 반드시 준수합니다.

ABAP CDS 커스텀 Text View 작성 패턴 — 4가지 필수 규칙 포함 
// 커스텀 Text View: Z-테이블 기반 마스터 데이터에만 적용
@ObjectModel.dataCategory: #TEXT          // ① #DIMENSION 아닌 #TEXT 선언
@EndUserText.label: 'Company Code Text'

define view entity ZASPN_I_CompanyCodeText
  as select from ZASPN_T_CompanyText as T
{
  key T.SPRAS  as Language,    // ② Language를 반드시 key에 포함
  key T.BUKRS  as CompanyCode,

  // ③ @Semantics.text: true — 텍스트 필드임을 Analytics Engine에 선언
  @Semantics.text: true
  @EndUserText.label: 'Company Name'
      T.BUTXT  as CompanyCodeName
  // ④ Text Association Cardinality는 [0..*] — 언어 수만큼 레코드 존재
}
커스텀 Text View 4가지 필수 규칙

① View 레벨에 @ObjectModel.dataCategory: #TEXT를 선언합니다. #DIMENSION이 아닙니다.
② Language 필드가 반드시 key여야 합니다. 언어 코드가 key에 없으면 SAC에서 텍스트가 중복 표시됩니다.
③ 실제 텍스트 필드에 @Semantics.text: true를 선언해야 SAC가 해당 필드를 텍스트로 인식합니다.
④ Dimension View에서 이 Text View로의 Association Cardinality는 [0..*]으로 선언합니다.

Dimension View에서 Text Association 선언 — 전체 패턴

ABAP CDS ZASPN_I_CompanyCode — Text Association 완성 패턴 
@Analytics.dataCategory: #DIMENSION
@Analytics.dimensionType: #STANDARD
@EndUserText.label: 'Company Code'
@AccessControl.authorizationCheck: #CHECK
@VDM.viewType: #BASIC

define view entity ZASPN_I_CompanyCode
  as select from I_CompanyCode as CC

  // ① Text Association: Cardinality [0..*] — 언어별 다중 레코드
  association [0..*] to ZASPN_I_CompanyCodeText as _Text
    on $projection.CompanyCode = _Text.CompanyCode

{
  // ② @ObjectModel.text.association: 텍스트 소스 Association 지정
  @ObjectModel.text.association: '_Text'
  key CC.CompanyCode,

      CC.Currency          as CompanyCodeCurrency,
      CC.FiscalYearVariant,
      CC.CountryChartOfAccounts,

  // ③ Association을 필드로 노출 — SAC가 텍스트를 자동 조회
      _Text
}
ASPN Group — 다국어 시나리오

ASPN Europe(법인 코드 3000)은 영어·독일어 텍스트가 모두 필요합니다. Text View의 Language key 덕분에 SAC 사용자 언어 설정에 따라 자동으로 적절한 텍스트가 표시됩니다. BUTXT: 'ASPN Europe GmbH'(독일어), 'ASPN Europe Ltd'(영어)가 언어 코드로 구분되어 저장됩니다.

4. Value Help — SAC 필터 선택 목록 연결

SAC에서 Dimension을 필터로 사용할 때, 사용자가 값을 직접 입력하는 대신 드롭다운 목록에서 선택할 수 있도록 하는 것이 Value Help입니다. Dimension View에 @Consumption.valueHelpDefinition을 선언하면 Fiori 및 SAC 필터 화면에 선택 목록이 자동 제공됩니다.

ABAP CDS Value Help 선언 — 이익센터 Dimension (ZASPN_I_ProfitCenter) 
@Analytics.dataCategory: #DIMENSION
@EndUserText.label: 'Profit Center'
@AccessControl.authorizationCheck: #CHECK

define view entity ZASPN_I_ProfitCenter
  as select from I_ProfitCenter as PC

  association [0..*] to I_ProfitCenterText as _Text
    on  $projection.ControllingArea = _Text.ControllingArea
    and $projection.ProfitCenter    = _Text.ProfitCenter
    and $projection.ValidityEndDate = _Text.ValidityEndDate

{
  // Value Help 선언: 이 Dimension 자신을 Value Help 소스로 지정
  @Consumption.valueHelpDefinition: [{
      entity: {
          name: 'ZASPN_I_ProfitCenter',   // Value Help 제공 View (자기 자신)
          element: 'ProfitCenter'           // 매핑할 필드명
      }
  }]
  @ObjectModel.text.association: '_Text'
  key PC.ControllingArea,
  key PC.ProfitCenter,
  key PC.ValidityEndDate,

      PC.ProfitCenterIsBlocked,
      PC.Segment,
      PC.CompanyCode,

      _Text
}
// WHERE 절: 유효 기간 내 이익센터만 표시
where PC.ValidityEndDate >= $session.system_date
  and PC.ProfitCenterIsBlocked = ''
$session.system_date

CDS 내장 세션 변수입니다. 런타임 시점의 시스템 날짜를 반환합니다. 유효 기간 필터(ValidityEndDate >= $session.system_date)에 활용하여 만료된 마스터 데이터를 Value Help 목록에서 자동 제외합니다.

📋 외부 Value Help — 별도 Search Help View를 참조하는 패턴 ▼

자기 자신을 Value Help 소스로 쓰는 것 외에, 별도로 설계된 Search Help View를 참조할 수도 있습니다.

ABAP CDS외부 Value Help 참조 패턴
// Query View의 파라미터 필드에서 외부 Value Help 참조
@Consumption.valueHelpDefinition: [{
    entity: {
        name: 'ZASPN_I_ProfitCenter',  // 외부 Dimension View 참조
        element: 'ProfitCenter'
    },
    additionalBinding: [{
        localElement: 'ControllingArea',   // 추가 필터 바인딩
        element: 'ControllingArea'
    }]
}]
P_ProfitCenter : prctr

additionalBinding을 통해 상위 필터(예: ControllingArea)와 연동하면 선택 목록이 자동으로 좁혀집니다. ASPN Group에서 Korea 관리 회계 영역의 이익센터만 필터링하는 데 활용합니다.

5. 완성 예시 — ZASPN_I_GLAccount (GL 계정 Dimension)

ASPN Holdings의 연결재무제표 분석에서 가장 핵심적인 Dimension인 GL 계정 Dimension을 Text Association · Value Help · 계층 연결 포인트를 모두 포함하여 완성합니다.

ABAP CDS ZASPN_I_GLAccount — 완성형 GL 계정 Dimension View 
// ── View 레벨 Annotation ──
@Analytics.dataCategory: #DIMENSION
@Analytics.dimensionType: #STANDARD
@EndUserText.label: 'GL Account'
@EndUserText.quickInfo: 'ASPN Group GL Account Dimension'
@AccessControl.authorizationCheck: #CHECK
@VDM.viewType: #BASIC
@ObjectModel.representativeKey: 'GLAccount'  // 대표 키 필드 명시

define view entity ZASPN_I_GLAccount
  as select from I_GLAccountInChartOfAccounts as GL

  // ① Text Association — 언어별 계정 명칭
  association [0..*] to I_GLAccountText as _Text
    on  $projection.ChartOfAccounts = _Text.ChartOfAccounts
    and $projection.GLAccount       = _Text.GLAccount

  // ② Hierarchy Association — 계정 계층 (Tab 04에서 생성)
  association [0..1] to ZASPN_H_GLAccountHierarchy as _Hierarchy
    on  $projection.ChartOfAccounts = _Hierarchy.ChartOfAccounts
    and $projection.GLAccount       = _Hierarchy.GLAccount

{
  // ── 키 필드 ──
  @ObjectModel.text.association: '_Text'
  key GL.ChartOfAccounts,

  @ObjectModel.text.association: '_Text'
  @Consumption.valueHelpDefinition: [{
      entity: { name: 'ZASPN_I_GLAccount', element: 'GLAccount' },
      additionalBinding: [{
          localElement: 'ChartOfAccounts', element: 'ChartOfAccounts'
      }]
  }]
  key GL.GLAccount,

  // ── 속성 필드 ──
  @EndUserText.label: 'Account Type'
      GL.GLAccountType,          // X:B/S, Y:P&L 구분 — ASPN 재무분석 핵심

      GL.AccountIsMarkedForDeletion,
      GL.IsBalanceSheetAccount,

  // ── ASPN 커스텀 분류 필드 ──
  @EndUserText.label: 'ASPN FS Item'
  case GL.GLAccountType
    when 'X' then 'Balance Sheet'
    when 'Y' then 'Income Statement'
    else          'Other'
  end                              as FinancialStatementType,

  // ── Association 노출 ──
  @ObjectModel.hierarchyAssociation: '_Hierarchy'
      _Text,
      _Hierarchy
}
where GL.AccountIsMarkedForDeletion = ''   // 삭제 표시 계정 제외
설계 포인트 — GLAccountType 분류

ASPN Holdings의 연결재무제표는 B/S와 P&L을 구분 분석해야 합니다. GLAccountType = 'X'(재무상태표 계정)와 'Y'(손익계산서 계정) 구분이 GL 계정 Dimension에 포함되어야 SAC에서 재무제표 유형별 필터가 가능합니다. 이 필드는 Cube View의 Restricted Measure 설계(Tab 06)에서도 핵심으로 활용됩니다.

6. Dimension View 설계 단계별 프로세스

1

분석 축(Dimension) 목록 확정

Cube View에서 사용할 키 필드 목록을 먼저 정의합니다. 각 키 필드가 하나의 Dimension으로 대응됩니다. ASPN Group 예시: CompanyCode, FiscalYear, FiscalPeriod, GLAccount, ProfitCenter, CostCenter, Currency 등.

2

소스 테이블 / Interface View 확인

SAP 표준 Interface View(I_CompanyCode, I_ProfitCenter 등)를 소스로 활용합니다. 표준 View가 없는 경우 Z 커스텀 테이블을 직접 참조합니다. 표준 View 재사용은 SAP 업그레이드 대응력을 높입니다.

3

Text View 확인 및 생성

표준 Text View(I_CompanyCodeText, I_GLAccountText)가 있으면 그대로 사용합니다. 커스텀 Dimension의 경우 @ObjectModel.dataCategory: #TEXT + @Semantics.text: true로 별도 Text View를 작성합니다. Language 필드를 반드시 key에 포함합니다.

4

Dimension View 작성

@Analytics.dataCategory: #DIMENSION 선언 → Text Association + @ObjectModel.text.association → Value Help 선언(@Consumption.valueHelpDefinition) → Hierarchy Association 포인트 예약(Tab 04) → 필요 속성 필드 추가 → WHERE 절로 유효 레코드 필터.

5

DCL(Access Control) 작성

Dimension View에도 반드시 DCL을 작성합니다. Value Help 조회 시에도 권한 필터가 동작하여 권한 없는 마스터 데이터가 선택 목록에 표시되지 않습니다. DCL 작성은 Tab 08에서 상세히 다룹니다.

6

Cube View에서 Foreign Key Association으로 참조

Cube View에서 @ObjectModel.foreignKey.association: '_CompanyCode'로 이 Dimension View를 참조합니다. SAC가 이 연결을 통해 Dimension을 자동 인식합니다 (Tab 05에서 계속).

B Dimension View의 View 레벨 Annotation 블록에
