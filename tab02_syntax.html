1. DDL 기반 선언 구조

CDS View는 ABAP Development Tools(ADT)의 .ddls 파일에 DDL 구문으로 작성합니다. SQL의 CREATE VIEW와 유사하지만 Annotation 블록, Association 선언, 파라미터 정의 등 CDS 고유 구성 요소가 추가됩니다.

기본 골격 — CDS View Entity

ABAP CDS · DDL ASPN_FinancialItem — 기본 골격 예시 
// ① Annotation 블록: 데이터 모델 메타데이터 선언
@AbapCatalog.viewEnhancementCategory: [#NONE]
@AccessControl.authorizationCheck: #CHECK
@EndUserText.label: 'ASPN Financial Item'
@VDM.viewType: #CONSUMPTION
@Analytics.dataCategory: #CUBE

// ② 파라미터 선언 (선택)
define view entity ZASPN_C_FinancialItem
  with parameters
    P_FiscalYear : fiscalyear

// ③ SELECT 소스
  as select from I_JournalEntryItem as JournalEntry

// ④ Association 선언 (지연 평가 — JOIN 아님)
  association [0..1] to I_CompanyCode as _CompanyCode
    on $projection.CompanyCode = _CompanyCode.CompanyCode

// ⑤ 필드 목록
{
  // key 필드 명시 (View Entity 필수)
  key JournalEntry.CompanyCode,
  key JournalEntry.FiscalYear,
  key JournalEntry.AccountingDocument,
  key JournalEntry.AccountingDocumentItem,

      JournalEntry.GLAccount,
      JournalEntry.AmountInCompanyCodeCurrency,
      JournalEntry.CompanyCodeCurrency,

  // Association을 필드로 노출 (소비자가 필요 시 JOIN)
      _CompanyCode
}
구성 요소 요약

① Annotation 블록 — @로 시작하는 메타데이터. 데이터 카테고리, 권한 체크, 라벨 등을 선언합니다.
② 파라미터 — with parameters로 런타임 입력값을 받습니다. Fiori / SAC Variable과 연동됩니다.
③ SELECT 소스 — 테이블 또는 다른 CDS View를 소스로 지정합니다.
④ Association — 조인 조건을 선언하되, 즉시 실행하지 않습니다 (지연 평가).
⑤ 필드 목록 — key 키워드 명시가 View Entity의 필수 요건입니다.

SQL CREATE VIEW vs. CDS View Entity — 선언 방식 비교

SQL 표준 SQL CREATE VIEW 
-- 메타데이터 선언 불가
-- 권한 체크 별도 구현 필요
-- 텍스트/라벨 정보 없음

CREATE VIEW V_FinancialItem AS
SELECT
    je.BUKRS   AS CompanyCode,
    je.GJAHR   AS FiscalYear,
    je.HKONT   AS GLAccount,
    je.DMBTR   AS Amount,
    cc.BUTXT   AS CompanyName
FROM  BSEG je
-- JOIN이 즉시 실행됨 (항상 비용 발생)
LEFT JOIN T001 cc
  ON je.BUKRS = cc.BUKRS;
ABAP CDS CDS View Entity — 동일 목적 
@EndUserText.label: 'Financial Item'
@AccessControl.authorizationCheck: #CHECK
@Semantics.amount.currencyCode: 'CompanyCodeCurrency'

define view entity ZASPN_C_FinancialItem
  as select from I_JournalEntryItem as je

  // Association: 필요 시에만 JOIN 실행
  association [0..1] to I_CompanyCode
    as _CompanyCode
    on $projection.CompanyCode
     = _CompanyCode.CompanyCode
{
  key je.CompanyCode,
  key je.FiscalYear,
      je.GLAccount,
      je.AmountInCompanyCodeCurrency,
      je.CompanyCodeCurrency,
      _CompanyCode  // 노출만, 실행 안 함
}

2. Annotation 우선 원칙

CDS View에서 Annotation은 단순 주석이 아니라 런타임 동작을 제어하는 선언입니다. SQL 힌트나 프로그램 로직 대신 Annotation으로 의미와 동작을 정의하는 것이 CDS 설계 철학의 핵심입니다.

Annotation 네임스페이스 대표 Annotation 역할 및 설명
@AbapCatalog @AbapCatalog.viewEnhancementCategory View 확장 허용 여부, SQL View 이름 설정 (구형 CDS View 전용)
@AccessControl @AccessControl.authorizationCheck: #CHECK DCL Access Control 적용 여부. #NOT_REQUIRED 설정 시 권한 체크 없이 전체 노출 — 운영 금지
@Analytics @Analytics.dataCategory
@Analytics.query		 데이터 카테고리(#DIMENSION / #CUBE) 지정. @Analytics.query: true는 Query View 선언의 핵심
@Semantics @Semantics.amount.currencyCode
@Semantics.quantity.unitOfMeasure		 금액 필드의 통화 코드 필드, 수량 필드의 단위 필드 연결. SAC에서 통화 변환 자동 처리에 필수
@DefaultAggregation @DefaultAggregation: #SUM
@DefaultAggregation: #MIN		 Measure 필드의 기본 집계 방식 선언. Analytics Engine이 자동 집계 시 사용
@EndUserText @EndUserText.label
@EndUserText.quickInfo		 SAC · Fiori UI에 표시되는 필드 레이블. 번역 가능 (SE63)
@VDM @VDM.viewType: #BASIC
@VDM.viewType: #CONSUMPTION		 VDM 계층 분류. Tab 01 계층 구조와 직접 연결
@ObjectModel @ObjectModel.text.association
@ObjectModel.foreignKey.association		 텍스트 Association 연결, 외래 키 Association 지정. Dimension View에서 핵심적으로 사용
@Consumption @Consumption.filter.selectionType
@Consumption.valueHelpDefinition		 필터 동작 방식 및 Value Help 연결. Query View의 Variable 설계에 사용
주의 — #NOT_REQUIRED 운영 절대 금지

@AccessControl.authorizationCheck: #NOT_REQUIRED는 개발·디버깅 전용입니다. SAC Live Model에 연결된 Query View에 이 설정이 남아 있으면, 권한 없는 사용자에게 전체 데이터가 노출됩니다. Tab 08에서 DCL 설계와 함께 반드시 #CHECK로 전환합니다.

View 레벨 vs. 필드 레벨 Annotation

ABAP CDS Annotation 적용 위치 구분 
// ── View 레벨 Annotation: View 전체에 적용 ──
@Analytics.dataCategory: #CUBE
@VDM.viewType: #CONSUMPTION
@AccessControl.authorizationCheck: #CHECK

define view entity ZASPN_C_FinancialCube
  as select from I_JournalEntryItem as je
{
  // ── 필드 레벨 Annotation: 해당 필드에만 적용 ──
  key je.CompanyCode,
  key je.FiscalYear,

  // 금액 필드: 통화 코드 필드와 연결
  @Semantics.amount.currencyCode: 'CompanyCodeCurrency'
  @DefaultAggregation: #SUM
  @EndUserText.label: 'Amount (CC Currency)'
      je.AmountInCompanyCodeCurrency,

  // 통화 코드 필드: Semantics 지정
  @Semantics.currencyCode: true
      je.CompanyCodeCurrency
}

3. Association vs. JOIN — 지연 평가(Lazy Evaluation)

CDS에서 가장 중요한 개념 중 하나입니다. Association은 JOIN 조건을 선언하지만 즉시 실행하지 않습니다. 소비자(상위 CDS View 또는 OData)가 Association 필드를 실제로 참조할 때만 JOIN이 실행됩니다.

구분 JOIN (SQL / CDS 인라인 JOIN) Association (CDS 권장)
실행 시점 SELECT 실행 시 즉시 JOIN 수행 소비자가 Association 필드 참조 시에만 실행 (지연 평가)
성능 항상 JOIN 비용 발생 불필요한 JOIN 제거 → 성능 유리
재사용성 하드코딩된 JOIN — 재사용 불가 Association을 노출하면 상위 View에서 선택적으로 사용
Cardinality 없음 [0..1], [1..1], [0..*] 명시 → 데이터 모델 명확화
사용 시나리오 항상 필요한 필드, 집계 기준 테이블 텍스트 조회, 마스터 데이터 참조, Dimension View 연결

Association 선언 및 소비 패턴

ABAP CDS Association 선언 → 노출 → 상위 View에서 소비 
// ① 하위 View — Association 선언 및 노출
define view entity ZASPN_I_JournalItem
  as select from I_JournalEntryItem as je

  // Cardinality [0..1]: CompanyCode 1개에 CompanyCode 마스터 0~1개
  association [0..1] to I_CompanyCode     as _CompanyCode
    on  $projection.CompanyCode = _CompanyCode.CompanyCode

  association [0..1] to I_GLAccountInChartOfAccounts as _GLAccount
    on  $projection.ChartOfAccounts = _GLAccount.ChartOfAccounts
    and $projection.GLAccount       = _GLAccount.GLAccount
{
  key je.CompanyCode,
  key je.FiscalYear,
      je.GLAccount,
      je.ChartOfAccounts,
      // Association을 필드로 노출 — 이 시점에서는 JOIN 미발생
      _CompanyCode,
      _GLAccount
}

// ② 상위 View — Association 소비 (이 시점에 JOIN 실행)
define view entity ZASPN_C_FinancialQuery
  as select from ZASPN_I_JournalItem as Item
{
  key Item.CompanyCode,
      // _CompanyCode Association을 통해 회사명 참조 → 이 때 JOIN 발생
      Item._CompanyCode.CompanyCodeName,
      Item.GLAccount
      // _GLAccount를 참조하지 않으면 GLAccount 테이블 JOIN 없음
}
$projection 키워드

$projection은 현재 View의 SELECT 필드 목록을 가리킵니다. Association ON 조건에서 자기 자신 필드를 참조할 때 사용합니다. 테이블 별칭(alias)과 혼동하지 않도록 주의합니다. $projection.CompanyCode는 이 View가 노출하는 CompanyCode 필드를 의미합니다.

4. CDS 고유 문법 요소

파라미터 ($parameters)

ABAP CDS 파라미터 선언 및 참조 — ASPN 회계연도 필터 
@EndUserText.label: 'ASPN Financial Query by Year'
@Analytics.query: true

define view entity ZASPN_C_FinancialQuery
  with parameters
    @Consumption.filter.selectionType: #SINGLE
    @Consumption.filter.mandatory: true
    P_FiscalYear  : fiscalyear,     // ABAP 데이터 타입 사용

    @Consumption.filter.selectionType: #SINGLE
    P_CompanyCode : bukrs            // 회사 코드 타입

  as select from ZASPN_I_JournalItem( // 하위 View에 파라미터 전달
      P_FiscalYear  = $parameters.P_FiscalYear,
      P_CompanyCode = $parameters.P_CompanyCode
  ) as Item
{
  key Item.CompanyCode,
  key Item.FiscalYear,
      Item.GLAccount,
      Item.AmountInCompanyCodeCurrency

  // WHERE 절에서 파라미터 사용
} where
    FiscalYear  = $parameters.P_FiscalYear
and CompanyCode = $parameters.P_CompanyCode

CASE 표현식 및 CAST

ABAP CDS CASE · CAST 활용 — ASPN 법인 분류 
{
  key CompanyCode,

  // CASE: 조건부 값 반환 (SQL CASE와 동일 구조)
  @EndUserText.label: 'Entity Type'
  case
    when CompanyCode = '1000' then 'Holdings'
    when CompanyCode = '2000' then 'Korea'
    when CompanyCode = '3000' then 'Europe'
    else                           'Associate'
  end                as EntityType,

  // CAST: 데이터 타입 변환 — 타입 명시 필수
  cast(AmountInCompanyCodeCurrency as abap.dec(23,2))
                     as AmountDecimal,

  // 산술 표현식
  AmountInGlobalCurrency * -1  as AmountNegated
}

NULL 처리 — COALESCE · NULLIF

ABAP CDS NULL 처리 Built-in Function 
{
  // COALESCE: 첫 번째 NOT NULL 값 반환 (SQL과 동일)
  coalesce(ProfitCenter, 'DEFAULT')  as ProfitCenterSafe,

  // NULLIF: 두 값이 같으면 NULL 반환
  nullif(CostCenter, 'DUMMY')         as CostCenterClean,

  // CASE로 NULL 대체 (명시적)
  case when SegmentForSAP is null
    then 'UNASSIGNED'
    else SegmentForSAP
  end                                  as SegmentSafe
}

currency_conversion — ASPN Group 통화 변환 예시

ABAP CDS EUR → KRW 변환 (ASPN Europe 연결 시나리오) 
{
  key CompanyCode,
  key FiscalYear,

  // 원본 EUR 금액
  @Semantics.amount.currencyCode: 'CompanyCodeCurrency'
  @DefaultAggregation: #SUM
      AmountInCompanyCodeCurrency    as AmountLC,

  // currency_conversion: SAP 환율 테이블 기반 통화 변환
  @Semantics.amount.currencyCode: 'GroupCurrency'
  @DefaultAggregation: #SUM
      currency_conversion(
          amount             => AmountInCompanyCodeCurrency,
          source_currency    => CompanyCodeCurrency,
          target_currency    => 'KRW',         // ASPN Holdings 그룹 통화
          exchange_rate_date => PostingDate,
          exchange_rate_type => 'M'            // 환율 유형: M=평균환율
      )                                        as AmountGC,

  'KRW'                                      as GroupCurrency,
      CompanyCodeCurrency
}

6. SQL 대비 주요 제약사항

CDS View는 SQL의 모든 기능을 지원하지 않습니다. 아래 제약사항은 SQL 배경 개발자가 CDS로 전환 시 가장 많이 실수하는 항목입니다. 제약을 우회해야 할 경우 AMDP(Tab 07)를 활용합니다.

✗

서브쿼리 (Subquery) 사용 불가

CDS View의 SELECT 필드 목록 및 WHERE 절에서 서브쿼리를 직접 사용할 수 없습니다. 복잡한 서브쿼리 로직은 별도 CDS View로 분리하거나 AMDP로 처리해야 합니다.

!

UNION / UNION ALL 제한적 지원

UNION ALL은 지원되지만 제약이 있습니다. 두 SELECT의 필드 수와 타입이 정확히 일치해야 하며, UNION된 View에는 Association을 선언할 수 없습니다. Hierarchy View에서는 UNION ALL이 금지됩니다.

✗

HANA Native Function 직접 호출 불가

HANA SQL의 Native Function(SERIES_GENERATE_DATE, HIERARCHY 등)을 CDS DDL에서 직접 호출할 수 없습니다. 이 경우 CDS Table Function + AMDP를 사용합니다 (Tab 07).

✗

DML 구문 사용 불가 (INSERT / UPDATE / DELETE)

CDS View는 읽기 전용 데이터 모델입니다. 데이터 변경은 ABAP RAP(Restful Application Programming Model)을 통해 구현합니다. SAC Planning Write-Back은 ACDOCP 테이블에 직접 기록하는 별도 메커니즘입니다.

!

GROUP BY — View 유형에 따라 사용 방식이 다름

일반 CDS View (Dimension, Interface): SQL과 동일하게 group by를 명시해야 집계 함수(sum, count 등)를 사용할 수 있습니다. group by 없이 집계 함수를 쓰면 활성화 오류가 발생합니다.

Analytics Cube View (#CUBE): group by를 작성하지 않습니다. 대신 @DefaultAggregation Annotation으로 각 Measure의 집계 방식을 선언하면 Analytics Engine이 런타임에 자동 집계합니다. Cube View에 group by를 명시하면 오류가 발생합니다.

!

HAVING 절 제약

HAVING은 CDS View에서 사용 가능하지만 Analytics Query View(@Analytics.query: true)에서는 사용할 수 없습니다. 집계 후 필터링이 필요한 경우 Calculated Measure의 Exception Aggregation을 활용합니다 (Tab 06 참조).

✗

순환 참조 (Circular Reference) 금지

CDS View A가 CDS View B를 참조하고, B가 다시 A를 참조하는 순환 구조는 컴파일 오류를 발생시킵니다. VDM 계층(Raw → Interface → Consumption) 단방향 참조 원칙을 반드시 준수합니다.

!

SELECT * 사용 비권장

SELECT *는 기술적으로 가능하지만 사용하지 않습니다. 소스 View의 필드 변경이 하위 View에 의도치 않게 전파되어 운영 장애로 이어질 수 있습니다. 명시적 필드 나열이 표준입니다.

📋 SQL 개발자를 위한 CDS 전환 체크리스트 ▼
SQL 습관CDS 전환 방식
서브쿼리로 집계 후 JOIN별도 CDS View로 분리 → 상위 View에서 Association
GROUP BY + SUM()@DefaultAggregation: #SUM Annotation 선언
