1. CDS Table Function과 AMDP — 개념 및 필요성

Tab 02에서 다뤘듯 CDS View DDL은 서브쿼리, 루프, HANA Native Function을 직접 지원하지 않습니다. 이를 해결하는 메커니즘이 CDS Table Function + AMDP 조합입니다. CDS DDL에서 함수 인터페이스를 선언하고, 실제 구현은 ABAP 클래스 메서드에 HANA SQL로 작성합니다.

CDS Table Function은 인터페이스(입출력 스펙)만 선언합니다. 실제 로직은 ABAP 클래스 메서드에 HANA SQLScript로 작성하며, 런타임에 HANA 엔진이 직접 실행합니다. ABAP 레이어를 거치지 않아 성능이 뛰어납니다.

2. AMDP 사용 적합 시나리오

ASPN Group 핵심 시나리오
기간별 누적 잔액 계산
재무상태표 계정의 기간별 누적 잔액은 전기이월 잔액 + 당기 발생액의 누적 합산입니다. 이 계산에는 기간 순서에 따른 누적 합산(Running Total)이 필요하며, HANA 윈도우 함수 SUM() OVER (PARTITION BY … ORDER BY …)로 구현합니다. CDS View DDL로는 윈도우 함수를 직접 사용할 수 없습니다.
데이터 전환
BW 마이그레이션 데이터 변환
BW에서 S/4HANA로 이관된 데이터는 구조가 다릅니다. 복수 BW DSO 데이터를 조인하고, 키 필드를 매핑하고, NULL을 처리하는 복잡한 변환 로직을 CDS View 단독으로 구현하기 어렵습니다. AMDP에서 SQLScript로 변환 후 Table Function으로 노출합니다.
원가 계산
다단계 원가 배부 계산
ASPN Korea의 간접비 배부 로직은 배부 기준(인원수, 면적)에 따라 여러 단계를 반복하며 원가를 분할합니다. 각 배부 단계가 이전 단계 결과에 의존하는 순차적 루프 계산은 CDS View로 표현 불가능하며 SQLScript LOOP로 구현합니다.
시계열 분석
날짜 시계열 생성 (SERIES_GENERATE)
특정 기간의 모든 날짜/월을 행으로 생성하는 HANA Native Function SERIES_GENERATE_DATE는 CDS DDL에서 직접 호출 불가합니다. ASPN Group의 월별 트렌드 분석에서 데이터가 없는 월도 0으로 표시하려면 이 함수로 기준 날짜 테이블을 생성 후 LEFT JOIN합니다.
AMDP 남용 주의

CDS View DDL로 해결 가능한 로직은 반드시 CDS View로 구현합니다. AMDP는 디버깅이 어렵고, ADT의 CDS 모델링 툴과 연동이 제한됩니다. 또한 ABAP 유닛 테스트 작성이 복잡합니다. Tab 02의 제약사항 목록에 해당하는 경우에만 AMDP를 선택합니다.

3. DEFINE TABLE FUNCTION 구문

CDS Table Function은 입력 파라미터와 반환 컬럼 스펙만 선언합니다. 실제 데이터 반환 로직은 implemented by method로 지정된 ABAP 클래스 메서드에 위임합니다.

ABAP CDS · DDL ZASPN_TF_PeriodBalance — Table Function 해부
1–4
@EndUserText.label: 'ASPN Period Balance Table Function' @ClientHandling.type: #CLIENT_INDEPENDENT @AccessControl.authorizationCheck: #NOT_REQUIRED // Table Function은 AccessControl을 직접 지원하지 않음 // → 소비하는 Cube View에서 권한 제어
Table Function Annotation. #CLIENT_INDEPENDENT: 클라이언트 필드 처리 방식. @AccessControl: #NOT_REQUIRED: Table Function 자체는 DCL 불가. 소비 View에서 #CHECK로 제어합니다.
5–10
define table function ZASPN_TF_PeriodBalance // 입력 파라미터 선언 with parameters P_FiscalYear : fiscalyear, P_CompanyCode : bukrs // ABAP 클래스 메서드를 구현체로 지정 implemented by method ZCL_ASPN_AMDP_BALANCE=>GET_PERIOD_BALANCE
핵심 선언. define table function: Table Function 선언 키워드. implemented by method: 구현 ABAP 클래스와 메서드를 클래스명=>메서드명 형식으로 지정합니다.
11–26
// 반환 컬럼 스펙 선언 (실제 데이터 로직 아님) returns { key CompanyCode : abap.char(4); key FiscalYear : abap.numc(4); key FiscalPeriod : abap.numc(3); key GLAccount : abap.char(10); key ProfitCenter : abap.char(10); // 집계 컬럼 — ABAP 타입으로 명시 CumulativeAmountLC : abap.curr(23,2); // 누적 잔액 (LC) CumulativeAmountGC : abap.curr(23,2); // 누적 잔액 (GC) PeriodAmountLC : abap.curr(23,2); // 당기 발생액 (LC) CompanyCodeCurrency : abap.cuky; // 통화 코드 GroupCurrency : abap.cuky; }
returns 블록. 반환 컬럼의 이름과 ABAP 타입을 선언합니다. 세미콜론(;)으로 끝납니다 (일반 CDS View와 달리). ABAP 타입(abap.char, abap.curr 등)을 직접 사용합니다. 이 스펙이 ABAP 클래스 메서드의 반환 테이블 타입과 일치해야 합니다.
returns 블록 — 세미콜론 주의

Table Function의 returns 블록에서 각 컬럼 선언은 일반 CDS View 필드 목록의 쉼표(,) 대신 세미콜론(;)으로 끝납니다. 또한 타입은 ABAP Dictionary 타입(abap.char, abap.numc, abap.curr, abap.cuky)으로 명시해야 합니다. DDIC 데이터 요소 참조(: bukrs 형식)는 사용 불가합니다.

4. ABAP 클래스 구현 — IF_AMDP_MARKER_HDB

AMDP는 ABAP 클래스로 구현합니다. 클래스가 IF_AMDP_MARKER_HDB 인터페이스를 구현(implements)하면 ABAP 런타임이 해당 클래스를 AMDP 클래스로 인식합니다. 메서드 본문에는 ABAP 코드 대신 HANA SQLScript를 작성합니다.

클래스 정의 — ZCL_ASPN_AMDP_BALANCE

ABAP ZCL_ASPN_AMDP_BALANCE — 클래스 정의부 (CLASS DEFINITION) 
CLASS ZCL_ASPN_AMDP_BALANCE DEFINITION
  PUBLIC
  FINAL
  CREATE PUBLIC.

  PUBLIC SECTION.

    "AMDP 클래스임을 선언하는 인터페이스 — 반드시 구현
    INTERFACES IF_AMDP_MARKER_HDB.

    "Table Function에서 implemented by method로 지정한 메서드
    CLASS-METHODS GET_PERIOD_BALANCE
      IMPORTING
        VALUE(IV_FISCAL_YEAR)  TYPE fiscalyear
        VALUE(IV_COMPANY_CODE) TYPE bukrs
      EXPORTING
        VALUE(ET_RESULT) TYPE STANDARD TABLE OF ZASPN_TT_PERIOD_BALANCE
                                    "반환 테이블 타입 — returns 블록과 일치
      RAISING
        CX_AMDP_ERROR.

ENDCLASS.
IMPORTING / EXPORTING 파라미터 매핑

IMPORTING 파라미터: Table Function의 with parameters와 이름·타입이 일치해야 합니다.
EXPORTING 파라미터: Table Function의 returns 블록과 구조가 일치하는 테이블 타입을 사용합니다.
타입 불일치 시 Table Function 활성화 오류가 발생합니다.

클래스 구현 — AMDP 메서드 본문 (SQLScript)

ABAP + SQLScript ZCL_ASPN_AMDP_BALANCE — GET_PERIOD_BALANCE 구현부 
CLASS ZCL_ASPN_AMDP_BALANCE IMPLEMENTATION.

  METHOD GET_PERIOD_BALANCE
    BY DATABASE FUNCTION              "AMDP 메서드 선언 키워드
    FOR HDB                            "대상 DB: HANA (HDB)
    LANGUAGE SQLSCRIPT                 "구현 언어: SQLScript
    OPTIONS READ-ONLY                  "읽기 전용 선언 (DML 불가)
    USING                              "참조할 HANA DB 오브젝트 명시
      ACDOCA                           "Universal Journal
      TCURR                            "환율 테이블
      ZASPN_T_CARRY_FORWARD.           "전기이월 커스텀 테이블

    -- ────────────────────────────────────────────
       SQLScript 본문 시작
       ABAP 코드가 아닌 HANA SQLScript 문법
    ─────────────────────────────────────────── --

    -- 1단계: 전기이월 잔액 (기초잔액)
    carry_forward = SELECT
        BUKRS          AS CompanyCode,
        GJAHR          AS FiscalYear,
        '000'          AS FiscalPeriod,   -- 기초잔액 기간 = 000
        HKONT          AS GLAccount,
        PRCTR          AS ProfitCenter,
        BALANCE_AMOUNT AS CumulativeAmountLC,
        'KRW'          AS GroupCurrency,
        WAERS          AS CompanyCodeCurrency
    FROM ZASPN_T_CARRY_FORWARD
    WHERE BUKRS  = :IV_COMPANY_CODE    -- 입력 파라미터 참조: :변수명
      AND GJAHR  = :IV_FISCAL_YEAR;

    -- 2단계: 당기 발생액 (ACDOCA)
    period_amounts = SELECT
        RBUKRS         AS CompanyCode,
        GJAHR          AS FiscalYear,
        POPER          AS FiscalPeriod,
        HKONT          AS GLAccount,
        PRCTR          AS ProfitCenter,
        SUM(HSL)       AS PeriodAmountLC,   -- 당기 발생액 합산
        RHCUR          AS CompanyCodeCurrency
    FROM ACDOCA
    WHERE RBUKRS = :IV_COMPANY_CODE
      AND GJAHR  = :IV_FISCAL_YEAR
      AND RLDNR  = '0L'               -- 원장: Leading Ledger
    GROUP BY RBUKRS, GJAHR, POPER, HKONT, PRCTR, RHCUR;

    -- 3단계: 기초잔액과 당기발생액 UNION ALL 후 누적 집계
    --        HANA 윈도우 함수로 기간별 Running Total 계산
    combined = SELECT * FROM :carry_forward
               UNION ALL
               SELECT CompanyCode, FiscalYear, FiscalPeriod,
                      GLAccount, ProfitCenter,
                      PeriodAmountLC, 'KRW',
                      CompanyCodeCurrency
               FROM :period_amounts;

    -- 4단계: 윈도우 함수로 기간별 누적 잔액 (CumulativeAmountLC) 계산
    et_result = SELECT
        CompanyCode, FiscalYear, FiscalPeriod,
        GLAccount, ProfitCenter,
        PeriodAmountLC,
        -- 윈도우 함수: 계정·이익센터·회사별로 기간 순서에 따라 누적 합산
        SUM(PeriodAmountLC)
          OVER (
            PARTITION BY CompanyCode, GLAccount, ProfitCenter
            ORDER BY     FiscalYear, FiscalPeriod
            ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
          )                                       AS CumulativeAmountLC,
        0.00               AS CumulativeAmountGC, -- GC는 별도 환산 (간략화)
        'KRW'              AS GroupCurrency,
        CompanyCodeCurrency
    FROM :combined
    WHERE FiscalPeriod != '000';  -- 기초잔액 행 제외 (누적에 이미 포함)

  ENDMETHOD.

ENDCLASS.
SQLScript 핵심 포인트

① 입력 파라미터 참조: SQLScript 내에서 ABAP IMPORTING 파라미터는 :IV_FISCAL_YEAR처럼 콜론(:)을 앞에 붙여 참조합니다.
② 중간 결과 변수: carry_forward =, period_amounts = 같이 변수에 SELECT 결과를 할당합니다. 참조 시 :carry_forward 형태.
③ 윈도우 함수: SUM() OVER (PARTITION BY … ORDER BY … ROWS BETWEEN …)는 순수 HANA SQL 문법이며 CDS DDL에서는 사용 불가합니다.
④ 최종 결과: et_result =에 할당된 값이 EXPORTING 파라미터로 반환됩니다.

5. USING 절 — DB 오브젝트 명시적 선언

AMDP 메서드가 SQLScript에서 참조하는 모든 DB 테이블·View·함수는 USING 절에 명시적으로 선언해야 합니다. 선언하지 않으면 런타임 오류가 발생합니다.

USING 대상 예시 설명
DDIC 투명 테이블 ACDOCA, BKPF, T001 SQLScript에서 직접 참조하는 SAP / 커스텀 테이블. 클라이언트 필드(MANDT) 포함 테이블은 WHERE MANDT = 클라이언트 조건 추가 필요
CDS View I_JournalEntryItem CDS View를 SQLScript 내에서 직접 참조 가능. 단, Table Function 자체를 재귀 참조 불가
다른 AMDP 메서드 ZCL_OTHER_AMDP=>METHOD SQLScript에서 다른 AMDP를 호출하는 경우 선언. 복잡한 계산 로직의 모듈화에 활용
환율 테이블 TCURR, TCURF SQLScript 내 직접 환율 조인 시 선언. CDS currency_conversion() 대신 직접 환율 계산이 필요한 경우
클라이언트 필드(MANDT) 주의

SQLScript에서 DDIC 테이블(예: ACDOCA)을 참조할 때 클라이언트 필드(MANDT)는 자동 필터되지 않습니다. 반드시 WHERE 조건에 MANDT = SESSION_CONTEXT('CLIENT')를 추가해야 멀티 클라이언트 환경에서 데이터가 혼용되지 않습니다. CDS View를 USING에 선언하면 클라이언트 필터가 자동 적용됩니다.

6. Cube View에서 Table Function 소비

완성된 Table Function은 일반 CDS View와 동일하게 Cube View의 as select from 소스로 참조합니다. 이후 Tab 05의 Cube View 설계 원칙이 그대로 적용됩니다.

ABAP CDS ZASPN_C_BalanceCube — Table Function을 소스로 참조하는 Cube View 
@Analytics.dataCategory: #CUBE
@EndUserText.label: 'ASPN Period Balance Cube'
@AccessControl.authorizationCheck: #CHECK  // Cube에서 권한 제어

define view entity ZASPN_C_BalanceCube
  with parameters
    P_FiscalYear  : fiscalyear,
    P_CompanyCode : bukrs

  // ★ Table Function을 소스로 참조 — 파라미터 전달
  as select from ZASPN_TF_PeriodBalance(
      P_FiscalYear  = $parameters.P_FiscalYear,
      P_CompanyCode = $parameters.P_CompanyCode
  ) as TF

  // Dimension FK Association — 일반 Cube와 동일하게 선언
  association [0..1] to ZASPN_I_CompanyCode as _CompanyCode
    on $projection.CompanyCode = _CompanyCode.CompanyCode

  association [0..1] to ZASPN_I_GLAccount   as _GLAccount
    on  $projection.GLAccount = _GLAccount.GLAccount

{
  @ObjectModel.foreignKey.association: '_CompanyCode'
  key TF.CompanyCode,
  key TF.FiscalYear,
  key TF.FiscalPeriod,

  @ObjectModel.foreignKey.association: '_GLAccount'
  key TF.GLAccount,
  key TF.ProfitCenter,

  // Measure 선언 — returns 블록의 컬럼을 Measure로 선언
  @Semantics.amount.currencyCode: 'CompanyCodeCurrency'
  @DefaultAggregation: #SUM
      TF.CumulativeAmountLC,

  @Semantics.amount.currencyCode: 'CompanyCodeCurrency'
  @DefaultAggregation: #SUM
      TF.PeriodAmountLC,

  @Semantics.currencyCode: true
      TF.CompanyCodeCurrency,

      _CompanyCode,
      _GLAccount
}

📋 Table Function + AMDP 개발 순서 체크리스트 ▼
1

반환 테이블 타입 생성 (SE11)

ABAP Dictionary에서 Table Function의 returns 블록과 일치하는 구조체·테이블 타입을 생성합니다. (ZASPN_TT_PERIOD_BALANCE)

2

ABAP 클래스 정의 작성

ADT에서 클래스를 생성하고 IF_AMDP_MARKER_HDB 인터페이스 추가, 메서드 시그니처를 Table Function의 파라미터·반환 타입에 맞게 선언합니다.

3

SQLScript 로직 구현

클래스 구현부에 BY DATABASE FUNCTION FOR HDB LANGUAGE SQLSCRIPT USING …으로 메서드를 선언하고 SQLScript를 작성합니다.
