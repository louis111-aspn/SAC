SAP_COM_0065 vs. Datasphere

SAP_COM_0065: S/4HANA Public Cloud에서 SAC를 HANA Live Connection으로 직접 연결하는 Communication Scenario. Datasphere 없이 CDS View를 SAC에 직접 노출합니다. 동일 SAP BTP 테넌트 내에서 동작합니다.

Datasphere: SAP 외 이기종 데이터(AWS RDS, Snowflake 등)를 통합하거나 복잡한 데이터 변환이 필요한 경우에만 추가로 도입합니다. 순수 S/4HANA CDS View 기반 Analytics에는 필요하지 않습니다.

2. OData 서비스 노출 — Query View를 SAC에 공개

SAC는 CDS View를 직접 인식하지 않고 OData 서비스를 통해 접근합니다. Analytics Query View를 OData 서비스로 노출하는 방법은 두 가지입니다.

방법 @OData.publish: true (자동) 수동 OData 서비스 등록
적용 위치 Query View의 View 레벨 Annotation SEGW(SAP Gateway Service Builder) 또는 /IWFND/MAINT_SERVICE
서비스명 생성 CDS View 이름 + _SRV 자동 생성 관리자가 직접 서비스명 지정
S/4HANA Public Cloud ✓ 표준 권장 방식 제한적 (관리형 환경)
S/4HANA On-Premise 가능 ✓ 세밀한 서비스 관리 가능
활성화 CDS View 활성화 시 자동 등록 /IWFND/MAINT_SERVICE에서 수동 활성화 필요

@OData.publish 선언 — ZASPN_C_PLQuery

ABAP CDS Query View에 OData 자동 등록 선언 
// @OData.publish: true → CDS View 이름 + _SRV로 OData 서비스 자동 생성
// 생성되는 서비스명: ZASPN_C_PLQUERY_SRV
@OData.publish: true

@Analytics.query: true
@EndUserText.label: 'ASPN P&L / B&S Integrated Query'
@AccessControl.authorizationCheck: #CHECK

define view entity ZASPN_C_PLQuery
  with parameters …
  as select from ZASPN_C_FinancialCube(…) as Cube
{ … }

// 활성화 후 확인:
// [Public Cloud] Fiori Launchpad → Communication Arrangements 앱에서 서비스 상태 확인
// [On-Premise]   /IWFND/MAINT_SERVICE → 서비스 필터: ZASPN_C_PLQUERY*
OData 서비스 활성화 확인 — 환경별 방법

S/4HANA Public Cloud: @OData.publish: true 선언 후 CDS View 활성화 시 서비스가 자동 등록됩니다. Fiori Launchpad의 Communication Arrangements 앱 또는 Display OData Services 앱에서 서비스 상태를 확인합니다. /IWFND/MAINT_SERVICE 트랜잭션은 Public Cloud에 존재하지 않습니다.

S/4HANA On-Premise: /IWFND/MAINT_SERVICE에서 서비스가 Active 상태인지 확인하고, Inactive라면 수동으로 활성화합니다.

3. SAP_COM_0065 — Communication Arrangement 설정

S/4HANA Public Cloud에서 SAC와의 Live Connection은 SAP_COM_0065 Communication Scenario를 통해 설정합니다. S/4HANA 측 설정과 SAC 측 설정 두 단계로 구성됩니다.

S/4HANA 측 — Communication Arrangement 생성

1

Communication User 확인

Tab 08에서 생성한 ZASPN_SAC_COMM Communication User를 사용합니다. SU01에서 User Type이 Communication User인지, 전체 법인 권한 PFCG Role이 할당되어 있는지 재확인합니다.

2

Communication System 생성

Fiori Launchpad → Communication Systems 앱에서 SAC 시스템을 등록합니다. SAC 테넌트 URL을 Host Name으로 입력합니다. 예: aspn-analytics.eu10.hanacloudservices.cloud.sap

3

Communication Arrangement 생성 — SAP_COM_0065

Fiori Launchpad → Communication Arrangements 앱에서 Scenario: SAP_COM_0065(Analytics — Live Data Connection from SAP Analytics Cloud)를 선택합니다. Communication System과 Communication User(ZASPN_SAC_COMM)를 연결합니다.

4

서비스 URL 확인

Communication Arrangement 생성 완료 후 Inbound Service URL을 복사합니다. 형식: https://[테넌트].s4hana.ondemand.com/sap/opu/odata/sap/[서비스명] 이 URL을 SAC Live Connection 설정에 사용합니다.

SAC 측 — Live Connection 설정

5

SAC System → Connections → Add Connection

SAC 관리 화면에서 System → Connections → Add Connection을 선택합니다. Connection Type: SAP S/4HANA를 선택합니다. (Live Data Connection to SAP S/4HANA 옵션)

6

Connection 정보 입력

Host: S/4HANA 테넌트 URL 입력.
Authentication Method: User Name and Password 또는 OAuth 선택.
User: ZASPN_SAC_COMM (Communication User).
Password: Communication User 비밀번호.

7

연결 테스트 및 저장

Test 버튼으로 연결 성공 여부를 확인합니다. 성공 시 초록색 체크 표시. 실패 시 Section 5의 운영 이슈 패턴을 참조합니다. 저장하면 Connection이 활성화됩니다.

On-Premise vs. Public Cloud 차이

S/4HANA Public Cloud: SAP_COM_0065 Communication Arrangement 사용. Fiori Launchpad에서 설정.
S/4HANA On-Premise: SAP Gateway(/IWFND/MAINT_SERVICE)에서 OData 서비스 활성화 후 SAC에서 직접 URL 입력. Communication Arrangement 불필요. SSL 인증서 관리가 추가로 필요할 수 있음.

4. Live Model Import — Dimension·Measure 매핑 검증

Connection 설정 완료 후 SAC에서 Live Model을 생성합니다. Import 후 Dimension과 Measure가 올바르게 인식되었는지 반드시 검증합니다.

Live Model Import 절차

1

SAC Modeler → New Model → Live Data Model

SAC Modeler에서 New Model → Live Data Model을 선택합니다. 생성한 S/4HANA Connection을 선택합니다.

2

Query View 검색 및 선택

OData 서비스 목록에서 ZASPN_C_PLQUERY_SRV를 검색합니다. Query View 이름 또는 @EndUserText.label로 검색 가능합니다. 'ASPN P&L / B&S Integrated Query'를 선택합니다.

3

Dimension 자동 인식 확인

Import 후 Dimensions 패널에서 아래 항목이 자동 인식되었는지 확인합니다.
✓ CompanyCode → ZASPN_I_CompanyCode 텍스트 표시
✓ GLAccount → ZASPN_I_GLAccount 텍스트 + 계층 표시
✓ ProfitCenter → ZASPN_I_ProfitCenter 텍스트 + 계층 표시
✓ FiscalYear, FiscalPeriod → 시간 Dimension 인식

4

Measure 자동 인식 확인

Import 후 Measures 패널에서 아래 항목 확인합니다.
✓ AmountGC, AmountLC → 금액 Measure (통화 Dimension 연결 확인)
✓ RevenueGC, CostOfSalesGC → Restricted Measure
✓ GrossProfitGC, GrossMarginPct → Calculated Measure
✓ BSAmountGC → Exception Aggregation Measure

5

Variable(Prompt) 확인

Live Model을 Story에서 사용 시 Variable이 Prompt 화면에 표시되는지 확인합니다. P_FiscalYear(필수), P_CompanyCode(선택), 기간 범위가 모두 표시되어야 합니다. Value Help 드롭다운이 동작하는지도 확인합니다.

Import 후 매핑 검증 — Pass / Fail 기준

✓ 정상 (Pass) 기준
  • Dimension 패널에 CompanyCode·GLAccount·ProfitCenter 등 키 필드 표시
  • 각 Dimension 옆에 텍스트(Description) 아이콘 표시
  • GLAccount·ProfitCenter Dimension에 계층(Hierarchy) 아이콘 표시
  • Measure 패널에 금액 Measure가 통화 Dimension과 연결된 상태로 표시
  • Restricted/Calculated Measure가 별도 구분으로 표시
  • Variable Prompt 화면에 모든 파라미터 표시 및 Value Help 동작
  • 데이터 미리보기에서 ASPN Group 재무 데이터 표시
✗ 문제 (Fail) 증상
  • Dimension이 모두 Measure로 표시됨 → @DefaultAggregation 과다 선언 의심
  • 텍스트 미표시 (코드만 표시) → @ObjectModel.text.association 또는 @Semantics.text 누락
  • Hierarchy 아이콘 없음 → @ObjectModel.hierarchyAssociation 누락
  • 금액 Measure에 통화 미연결 → @Semantics.amount.currencyCode 누락
  • 데이터 공백 → DCL 오류 또는 Communication User 권한 미설정
  • Query View 자체가 목록에 미표시 → OData 서비스 비활성 또는 @Analytics.query: true 미선언

5. 운영 이슈 패턴 — 원인과 해결

🔴 Measure 미인식 — Dimension으로 표시
금액·수량 필드가 SAC에서 Measure가 아닌 Dimension으로 분류됩니다.
원인
해당 필드에 @DefaultAggregation Annotation 누락.
해결
Cube View에서 해당 숫자 필드에 @DefaultAggregation: #SUM 추가 후 재활성화. SAC Live Model 새로고침.
🔴 텍스트 미표시 — 코드만 표시
SAC에서 회사 코드(1000)가 회사명(ASPN Holdings)으로 표시되지 않습니다.
원인 3가지 중 하나
① Dimension View에 @ObjectModel.text.association 누락
② Text View에 @Semantics.text: true 누락
③ Text Association Cardinality가 [0..1] (다국어 환경)
해결
위 세 항목을 순서대로 확인·수정 후 재활성화.
🔴 Hierarchy 미로딩 — 드릴다운 불가
SAC에서 이익센터 Dimension을 행에 배치해도 계층형 드릴다운이 동작하지 않습니다.
원인 2가지
① Dimension View 필드 목록에서 @ObjectModel.hierarchyAssociation: '_Hierarchy' 누락
② Hierarchy View(.ddlx)가 활성화되지 않았거나 소스 View에 데이터 없음
해결
Annotation 추가 후 Dimension View 재활성화. 소스 View 데이터 유효성 확인.
🔴 데이터 공백 — 아무것도 표시 안 됨
SAC에서 Live Model 연결은 성공하지만 데이터가 전혀 표시되지 않습니다.
원인 체크 순서
① Communication User의 PFCG Role에 조회 대상 회사 코드 권한 없음
② DCL이 잘못 작성되어 모든 행이 필터됨 (ACTVT = '01' 오류 등)
③ Variable 필수값(mandatory: true)이 입력되지 않아 쿼리 미실행
④ WHERE 절 조건이 실제 데이터와 불일치 (예: FiscalYear >= '2030')
🟡 성능 이슈 — 쿼리 응답 지연
SAC에서 Live Model 데이터 로딩이 30초 이상 소요됩니다.
주요 원인
① 필수 Variable 미설정으로 전체 ACDOCA 스캔
② Cube View에서 과다한 JOIN (불필요한 Association 즉시 실행)
③ AMDP SQLScript 미최적화 (인덱스 미활용)
해결
필수 Variable 추가. 불필요한 인라인 JOIN을 Association으로 전환. HANA SQL Plan 분석.
🟡 Query View 목록 미표시
SAC에서 Live Model 생성 시 Connection은 성공하지만 Query View가 목록에 없습니다.
체크 순서
① @Analytics.query: true 선언 여부
② @OData.publish: true 선언 여부
③ [On-Premise] /IWFND/MAINT_SERVICE에서 서비스 활성화 상태 (Active vs. Inactive)
③ [Public Cloud] Fiori의 Display OData Services 앱에서 서비스 등록 상태 확인
④ Communication User의 S_SERVICE 권한에 서비스명 포함 여부
