1. @Analytics.query: true — Cube와의 결정적 차이

Query View는 Cube View를 소스로 하여 그 위에 사용자 목적에 맞는 분석 레이어를 추가합니다. SAC Live Model은 Query View만 직접 참조합니다. @Analytics.query: true 하나가 이 View를 Cube와 완전히 다른 오브젝트로 만듭니다.

항목 Cube View (#CUBE) Query View (@Analytics.query: true)
핵심 선언 @Analytics.dataCategory: #CUBE @Analytics.query: true
(dataCategory 선언 없음)
SAC 노출 직접 노출 안 됨 OData 서비스 등록 후 SAC Live Model 직접 참조
소스 트랜잭션 테이블 / Interface View 반드시 Cube View (#CUBE)를 소스로
Variable 파라미터만 가능 Variable 선언 — SAC 필터 화면과 직접 연동
Restricted Measure 선언 불가 선언 가능 — 특정 Dimension 조건으로 Measure 제한
Calculated Measure 선언 불가 선언 가능 — 다른 Measure 간 계산식 정의
Axis 설정 해당 없음 Row / Column / Free 기본 배치 설정
Exception Aggregation 해당 없음 집계 예외 처리 — B/S 잔액, 재고 수량 등에 활용
설계 실수 — Query View를 Cube 소스 외에서 참조

Query View의 소스(as select from)는 반드시 @Analytics.dataCategory: #CUBE가 선언된 Cube View여야 합니다. Interface View나 일반 CDS View를 직접 소스로 삼으면 Variable·Restricted Measure 등 Query 전용 기능이 정상 동작하지 않거나 활성화 오류가 발생합니다.

2. Variable — SAC 필터 화면 연동

Variable은 Query View의 파라미터(with parameters)에 @AnalyticsDetails.query.variable Annotation을 추가하여 선언합니다. Cube View의 with parameters와 달리 SAC 필터 화면(Prompt)에 자동으로 표시되어 사용자가 런타임에 값을 입력·선택합니다.

단일값 입력
Single Value Variable
selectionType: #SINGLE
SAC 필터 화면에서 하나의 값만 선택. 필수 입력 시 mandatory: true. 회계연도 단일 선택에 사용.
복수값 선택
Multiple Value Variable
selectionType: #MULTIPLE
복수 회사 코드 동시 선택. SAC 멀티 필터와 연동. Value Help(valueHelpDefinition)와 함께 사용하면 드롭다운 목록 제공.
구간 선택
Range Variable
selectionType: #RANGE
From~To 범위 입력. 회계기간 범위 필터에 사용. SAC 날짜 범위 필터 UI와 자동 연동.
기본값 자동 설정
Default Value Variable
@AnalyticsDetails.query.variableDefault
SAC 최초 로딩 시 기본값 자동 적용. 현재 회계연도를 기본값으로 설정하는 데 활용. defaultValue 또는 defaultValueFunction 사용.
입력 불필요
Optional Variable
mandatory: false
사용자 입력 없어도 Query 실행 가능. 입력 시 필터 적용, 미입력 시 전체 데이터. 주의: 대용량 Cube에서 미입력 시 성능 이슈 발생 가능.
세션 변수
Customer Exit Variable
variableType: #CUSTOMER_EXIT
ABAP Customer Exit으로 값을 동적 계산. 현재 사용자의 회사 코드 자동 설정 등에 활용. S/4HANA Public Cloud에서는 제한적.

Variable 선언 코드 — ASPN P&L Query

ABAP CDS Variable 선언 — 회계연도 · 회사 코드 · 회계기간 범위 
@Analytics.query: true
@EndUserText.label: 'ASPN P&L Query'
@AccessControl.authorizationCheck: #CHECK

define view entity ZASPN_C_PLQuery
  with parameters

    // ① Single Value Variable — 회계연도 (필수)
    @EndUserText.label: 'Fiscal Year'
    @AnalyticsDetails.query.variable: {
        selectionType: #SINGLE,
        mandatory:     true
    }
    P_FiscalYear  : fiscalyear,

    // ② Multiple Value Variable — 회사 코드 (선택, Value Help 연결)
    @EndUserText.label: 'Company Code'
    @AnalyticsDetails.query.variable: {
        selectionType: #MULTIPLE,
        mandatory:     false
    }
    @Consumption.valueHelpDefinition: [{
        entity: { name: 'ZASPN_I_CompanyCode', element: 'CompanyCode' }
    }]
    P_CompanyCode : bukrs,

    // ③ Range Variable — 회계기간 From~To
    @EndUserText.label: 'Fiscal Period From'
    @AnalyticsDetails.query.variable: {
        selectionType: #RANGE,
        mandatory:     false
    }
    P_FiscalPeriodFrom : fiscalmonth,

    @EndUserText.label: 'Fiscal Period To'
    @AnalyticsDetails.query.variable: {
        selectionType: #RANGE,
        mandatory:     false
    }
    P_FiscalPeriodTo  : fiscalmonth

  as select from ZASPN_C_FinancialCube( // Cube에 파라미터 전달
      P_FiscalYear  = $parameters.P_FiscalYear
  ) as Cube
Variable vs. 파라미터 — 중요한 차이

with parameters + @AnalyticsDetails.query.variable = Variable → SAC 필터 Prompt 화면에 표시
with parameters만 선언 (Annotation 없음) = 파라미터 → SAC UI에 표시되지 않음, 프로그램 방식 전달만 가능

Query View에서 사용자가 SAC 화면에서 직접 입력해야 하는 값은 반드시 Variable로 선언해야 합니다.

3. Restricted Measure와 Calculated Measure

특정 조건으로 범위 제한
Restricted Measure
정의: Cube의 Measure를 특정 Dimension 조건으로 필터링한 새로운 Measure.

선언 위치: Query View 필드 목록 — @AnalyticsDetails.query.selection

ASPN 예시: 전체 금액 Measure에서 손익계산서 계정(GLAccountType = 'Y')만 집계하는 "영업이익" Measure, Korea 법인(CompanyCode = '2000')만 필터링한 "Korea Revenue" Measure.

특징: 집계 방식은 Cube의 @DefaultAggregation을 상속. 조건은 Dimension 값으로만 제한 가능.
다른 Measure 간 계산
Calculated Measure
정의: Query View 내 다른 Measure(원본 또는 Restricted)를 조합·계산하는 파생 Measure.

선언 위치: Query View 필드 목록 — @AnalyticsDetails.query.formula

ASPN 예시: 매출액 − 매출원가 = 매출총이익, (영업이익 ÷ 매출액) × 100 = 영업이익률, 전기 대비 증감률.

특징: 사칙연산, CASE 표현식, 집계 함수 조합 가능. Exception Aggregation과 함께 활용 시 B/S 잔액 계산이 가능.

Restricted Measure 선언 — ASPN P&L

ABAP CDS Restricted Measure — 손익계산서 계정 필터 Measure 
{
  key Cube.CompanyCode,
  key Cube.FiscalYear,
  key Cube.FiscalPeriod,
  key Cube.GLAccount,
  key Cube.ProfitCenter,

  // ── 원본 Measure (Cube에서 상속) ──
      Cube.AmountLC,
      Cube.AmountGC,

  // ── Restricted Measure ① : 매출 (GLAccountType = 'Y' + 특정 계정 범위) ──
  @EndUserText.label: 'Revenue (GC)'
  @AnalyticsDetails.query.selection: [
      { dimension: 'FinancialStatementType', range: [{
          sign:     #INCLUDE,
          operator: #EQ,
          low:      'Income Statement'
      }]},
      { dimension: 'GLAccount', range: [{
          sign:     #INCLUDE,
          operator: #BT,   // BT = Between
          low:      '4000000',
          high:     '4999999'  // ASPN 매출 계정 범위
      }]}
  ]
      Cube.AmountGC  as RevenueGC,

  // ── Restricted Measure ② : 매출원가 ──
  @EndUserText.label: 'Cost of Sales (GC)'
  @AnalyticsDetails.query.selection: [
      { dimension: 'GLAccount', range: [{
          sign:     #INCLUDE,
          operator: #BT,
          low:      '5000000',
          high:     '5999999'
      }]}
  ]
      Cube.AmountGC  as CostOfSalesGC

Calculated Measure 선언 — 매출총이익·이익률

Calculated Measure Formula 참조 방식

@AnalyticsDetails.query.formula의 수식에서 다른 Measure를 참조할 때는 해당 Measure의 필드 별칭(alias)을 사용합니다. SAP 표준 CDS Query Calculated Measure에서 Measure 참조 표기는 구현 환경(BW/S4 Analytics Engine 버전)에 따라 $$MeasureName 또는 필드명 직접 참조 방식이 혼용됩니다. 아래 예시는 개념적 표현이며, 실제 프로젝트에서는 ADT의 CDS Annotation 자동완성과 SAP 공식 가이드를 기준으로 사용 환경의 정확한 구문을 확인하고 적용합니다.

ABAP CDS Calculated Measure — 매출총이익 · 영업이익률 계산 (개념 예시) 
  // ── Calculated Measure ① : 매출총이익 = 매출 − 매출원가 ──
  @EndUserText.label: 'Gross Profit (GC)'
  @DefaultAggregation: #SUM
  @Semantics.amount.currencyCode: 'GroupCurrency'
  // Formula: 다른 Measure 필드명을 참조하여 계산식 정의
  @AnalyticsDetails.query.formula: 'RevenueGC - CostOfSalesGC'
      Cube.AmountGC  as GrossProfitGC,

  // ── Calculated Measure ② : 매출총이익률 (%) ──
  @EndUserText.label: 'Gross Profit Margin (%)'
  @DefaultAggregation: #NOP   // 비율 — 합산 불가, 상위 레벨 재계산
  @AnalyticsDetails.query.formula: 'GrossProfitGC / RevenueGC * 100'
      Cube.AmountGC  as GrossMarginPct,

  // ── Calculated Measure ③ : 이익/손실 부호 분류 ──
  @EndUserText.label: 'P&L Status'
  @DefaultAggregation: #NOP
  @AnalyticsDetails.query.formula: 'GrossProfitGC GT 0 ? 1 : -1'
      Cube.AmountGC  as PLStatus
이익률 Measure의 @DefaultAggregation: #NOP

매출총이익률처럼 비율로 계산되는 Calculated Measure는 합산하면 의미가 없습니다. @DefaultAggregation: #NOP을 선언하여 Analytics Engine이 집계 방식을 위임받도록 합니다. SAC에서 이 Measure를 테이블에 표시할 때 집계 로직은 Formula에 정의된 대로 상위 레벨에서 재계산됩니다.

4. Exception Aggregation — B/S 잔액·재고 수량 처리

ASPN Group의 재무상태표(B/S) 금액은 기간별 합산이 아닌 특정 시점 잔액이어야 합니다. 예를 들어 현금 계정의 1분기 금액과 2분기 금액을 합산하면 실제 현금 잔액이 아닙니다. 이런 경우 Exception Aggregation을 사용하여 집계 예외 동작을 지정합니다.

Exception Aggregation 동작 ASPN 활용 시나리오
#LAST 참조 특성의 마지막 값 기준 집계 B/S 계정 잔액: 기간 범위 선택 시 마지막 기간 잔액 표시. 재고 수량 마감 잔액.
#FIRST 참조 특성의 첫 번째 값 기준 집계 기초 잔액(기간 시작 시점 잔액). 비교 분석용 전기 초 잔액.
#MAX 참조 특성 범위 내 최댓값 기간 중 최고 현금 잔액. 피크 재고 수량 분석.
#MIN 참조 특성 범위 내 최솟값 기간 중 최저 유동성 분석.
#COUNT_DISTINCT_NULLS 참조 특성의 고유값 수 거래 법인 수, 활성 이익센터 수 등 집계.
#AVG 참조 특성 범위 내 평균 월평균 현금 잔액, 분기 평균 재고.

Exception Aggregation 선언 — B/S 잔액 Measure

ABAP CDS B/S 잔액 Measure — Exception Aggregation #LAST 활용 
  // ── B/S 계정 잔액 — 기간별 합산 아닌 마지막 기간 잔액 ──
  @EndUserText.label: 'Balance Sheet Amount (GC)'
  @DefaultAggregation: #SUM
  @Semantics.amount.currencyCode: 'GroupCurrency'

  // Exception Aggregation: FiscalYearPeriod 기준으로 마지막 기간 값 사용
  @AnalyticsDetails.query.exceptionAggregationSteps: [{
      exceptionAggregationBehavior: #LAST,
      exceptionAggregationElements:  [ 'FiscalYearPeriod' ]
                          // FiscalYearPeriod 기준으로 마지막 기간 잔액 추출
  }]

  @AnalyticsDetails.query.selection: [
      { dimension: 'FinancialStatementType', range: [{
          sign:     #INCLUDE,
          operator: #EQ,
          low:      'Balance Sheet'     // B/S 계정만 (GLAccountType = 'X')
      }]}
  ]
      Cube.AmountGC  as BSAmountGC,

  // ── P&L 금액은 Exception Aggregation 없이 일반 #SUM ──
  @EndUserText.label: 'P&L Amount (GC)'
  @DefaultAggregation: #SUM
  @Semantics.amount.currencyCode: 'GroupCurrency'
  @AnalyticsDetails.query.selection: [
      { dimension: 'FinancialStatementType', range: [{
          sign:     #INCLUDE,
          operator: #EQ,
          low:      'Income Statement'
      }]}
  ]
      Cube.AmountGC  as PLAmountGC
ASPN Group — B/S vs. P&L 집계 구분 설계

ASPN Holdings 연결재무제표에서 재무상태표(B/S) 계정과 손익계산서(P&L) 계정은 집계 방식이 근본적으로 다릅니다. 하나의 Query View에서 BSAmountGC(Exception Aggregation #LAST)와 PLAmountGC(일반 #SUM) 두 Measure를 함께 선언하면 SAC에서 재무상태표와 손익계산서를 하나의 Live Model로 통합 분석할 수 있습니다. FinancialStatementType은 Tab 03 GL 계정 Dimension에서 CASE 구문으로 정의한 필드입니다.

5. Axis 설정 — SAC 기본 레이아웃 제어

@AnalyticsDetails.query.axis로 SAC에서 Live Model Import 시 각 필드가 기본적으로 어디에 배치될지를 지정합니다. SAC 사용자가 이후 드래그로 변경할 수 있지만, 초기 배치를 설계 의도에 맞게 미리 설정합니다.

Row Axis

행 (Row)

테이블의 행에 기본 배치. Dimension 분석 기준이 되는 필드에 사용. ASPN 예시: GL 계정, 이익센터.

@AnalyticsDetails.query.axis: #ROWS
Column Axis

열 (Column)

테이블의 열에 기본 배치. 비교 기준이 되는 필드에 사용. ASPN 예시: 회계기간, Measure.

@AnalyticsDetails.query.axis: #COLUMNS
Free Axis

자유 (Free)

초기 배치 없음 — 사용자가 드래그하여 배치. 필터 영역 기본 배치. ASPN 예시: 회계연도, 회사 코드.

@AnalyticsDetails.query.axis: #FREE

Axis 선언 코드 — ASPN P&L Query 레이아웃

ABAP CDS Axis 설정 — P&L Query SAC 기본 레이아웃 
{
  // ── FREE: 필터 영역 기본 배치 ──
  @AnalyticsDetails.query.axis: #FREE
  key Cube.CompanyCode,

  @AnalyticsDetails.query.axis: #FREE
  key Cube.FiscalYear,

  // ── ROWS: 행 기본 배치 ──
  @AnalyticsDetails.query.axis: #ROWS
  key Cube.GLAccount,

  @AnalyticsDetails.query.axis: #ROWS
  key Cube.ProfitCenter,

  @AnalyticsDetails.query.axis: #ROWS
  key Cube.ChartOfAccounts,

  // ── COLUMNS: 열 기본 배치 — 회계기간 ──
  @AnalyticsDetails.query.axis: #COLUMNS
  key Cube.FiscalPeriod,

  // ── Measure 필드 (Axis 선언 없음 — Measure는 자동으로 COLUMNS 배치) ──
      Cube.AmountGC,
      RevenueGC,
      CostOfSalesGC,
      GrossProfitGC,
      GrossMarginPct,
      BSAmountGC,
      PLAmountGC,

  // ── 통화·단위 필드 ──
      Cube.GroupCurrency,
      Cube.CompanyCodeCurrency
}

6. ZASPN_C_PLQuery — 완성형 전체 구조

Variable·Restricted Measure·Calculated Measure·Exception Aggregation·Axis를 모두 포함한 완성형 P&L Query View 전체 골격입니다.

ABAP CDS ZASPN_C_PLQuery — Analytics Query View 완성 구조
1–5
@Analytics.query: true @EndUserText.label: 'ASPN P&L / B&S Integrated Query' @AccessControl.authorizationCheck: #CHECK @VDM.viewType: #CONSUMPTION @OData.publish: true // OData 서비스 자동 등록 (Tab 09)
View 레벨 핵심 Annotation. @Analytics.query: true가 이 View를 Query View로 선언. @OData.publish: true는 SAC 연결을 위한 OData 서비스 자동 등록 (Tab 09 상세).
6–22
define view entity ZASPN_C_PLQuery with parameters // [Variable 선언 — Section 2 참조] P_FiscalYear : fiscalyear, P_CompanyCode : bukrs, P_FiscalPeriodFrom : fiscalmonth, P_FiscalPeriodTo : fiscalmonth as select from ZASPN_C_FinancialCube( P_FiscalYear = $parameters.P_FiscalYear ) as Cube
소스는 반드시 Cube View. 파라미터를 Cube에 전달합니다. P_CompanyCode와 기간 파라미터는 WHERE 절에서 필터링.
23–60
{ // ── 키 필드 + Axis + FK Association ── @AnalyticsDetails.query.axis: #FREE key Cube.CompanyCode, @AnalyticsDetails.query.axis: #FREE key Cube.FiscalYear, @AnalyticsDetails.query.axis: #ROWS key Cube.GLAccount, @AnalyticsDetails.query.axis: #ROWS key Cube.ProfitCenter, @AnalyticsDetails.query.axis: #COLUMNS key Cube.FiscalPeriod, key Cube.ChartOfAccounts, // ── 원본 Measure ── Cube.AmountGC, // ── Restricted Measure ── // [RevenueGC, CostOfSalesGC — Section 3 참조] // ── Calculated Measure ── // [GrossProfitGC, GrossMarginPct — Section 3 참조] // ── Exception Aggregation Measure ── // [BSAmountGC #LAST, PLAmountGC #SUM — Section 4 참조] // ── 통화 코드 ── Cube.GroupCurrency, Cube.CompanyCodeCurrency } // WHERE 절 — 파라미터 기반 필터 where Cube.CompanyCode in $parameters.P_CompanyCode and Cube.FiscalPeriod between $parameters.P_FiscalPeriodFrom and $parameters.P_FiscalPeriodTo
전체 필드 목록 구조. 키 필드에 Axis 선언 → 원본 Measure → Restricted Measure → Calculated Measure → Exception Aggregation Measure → 통화 코드 순서로 배치하는 것이 가독성 표준. WHERE 절에서 Multiple Variable(in)과 Range Variable(between) 활용.
