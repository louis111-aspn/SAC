1. CDS View란 무엇인가

정의

CDS(Core Data Services) View는 ABAP Dictionary View를 대체하는 데이터 모델링 언어입니다. SQL 기반 DDL(Data Definition Language)로 작성하며, Annotation을 통해 의미(Semantics)와 동작(Behavior)을 선언적으로 정의합니다.

구분 ABAP Dictionary View CDS View (DDIC-based) CDS View Entity (권장)
선언 위치 SE11 / DDIC ABAP Development Tools (ADT) ADT — .ddls 파일
구문 GUI 기반 DEFINE VIEW … AS SELECT DEFINE VIEW ENTITY … AS SELECT
Association 불가 가능 가능 + 향상된 타입 안정성
Annotation 불가 가능 가능 (전체 지원)
SAC Live 연결 불가 제한적 ✓ 완전 지원
Access Control (DCL) 불가 가능 ✓ 완전 지원
주의

신규 개발 시 ABAP Dictionary View 또는 구형 DEFINE VIEW 구문 사용은 지양합니다. SAP는 DEFINE VIEW ENTITY 방식을 표준으로 권장합니다.

CDS View vs. CDS View Entity — 핵심 차이점

두 구문은 외형이 유사하지만 키 명시 방식, 타입 안정성, Cardinality 강제 여부에서 본질적 차이가 있습니다. 현장에서는 신규 Consumption Layer를 DEFINE VIEW ENTITY로 개발하면서, SAP 표준 Interface View(I_*)를 Association 소스로 혼용하는 패턴이 일반적입니다.

현장 패턴 — ASPN Group 프로젝트

ASPN Holdings의 재무분석 Query View 개발 시, SAP 표준 Interface View인 I_JournalEntryItem(DEFINE VIEW 기반)을 Association 소스로 참조하면서, 신규 Cube·Query View는 DEFINE VIEW ENTITY로 작성합니다. 두 구문의 혼용은 허용되며, 이것이 S/4HANA 프로젝트의 표준 접근 방식입니다.

2. S/4HANA Virtual Data Model (VDM) 계층 구조

SAP S/4HANA의 VDM은 데이터를 목적에 따라 3계층으로 분리합니다. Analytics Query CDS View는 최상위 Consumption Layer에 위치하며, 하위 계층의 데이터를 집약·가공하여 SAC에 노출합니다.

1

Raw / Basic Interface Layer — 원천 데이터

데이터베이스 테이블을 직접 참조합니다. 비즈니스 의미 부여 없이 Raw 컬럼을 노출합니다.

I_* prefix @VDM.viewType: #BASIC ACDOCA, BKPF, BSEG
↓
2

Interface / Composite Layer — 비즈니스 의미 부여

여러 Basic View를 조합하여 비즈니스 엔티티(전표, 원가요소 등)를 구성합니다. Reuse 가능한 공통 구조를 제공합니다.

I_*, R_* prefix @VDM.viewType: #COMPOSITE @VDM.viewType: #TRANSACTIONAL
↓
3

Consumption Layer — Analytics Query (이 모듈의 핵심)

UI, Fiori, SAC, OData 등 최종 소비자에게 데이터를 노출합니다. Cube View → Analytics Query View 구조로 SAC Live Model에 연결됩니다.

C_* prefix @Analytics.query: true @Analytics.dataCategory: #CUBE @VDM.viewType: #CONSUMPTION
ASPN Group 시나리오 예시

ASPN Holdings의 연결재무제표 분석을 위해 ACDOCA(원천) → I_JournalEntryItem(Interface) → C_FinancialStatementCube(Cube) → C_FinancialStatementQuery(Query) 계층으로 구성합니다. SAC Live Model은 Query View만 직접 참조합니다.
